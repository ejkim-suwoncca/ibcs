// 01 · Part 3.1

Part 3.1:
웹사이트 구현

네비게이션과 레이아웃을 갖춘 완성된 웹사이트를 제출합니다. (10점)

필수 ①
네비게이션 + 페이지 레이아웃 — 여러 페이지로 이동할 수 있는 네비게이션 바와 각 페이지의 레이아웃이 완전히 구현되어야 합니다.
필수 ②
프론트엔드 · 백엔드 연결 — HTML/CSS/JavaScript(프론트엔드)와 서버 또는 데이터 처리(백엔드)가 유기적으로 연결되어야 합니다. 데이터 흐름이 논리적이어야 합니다.
필수 ③
의사코드 주석 — 코드 각 영역(함수, 처리 블록 등)에 해당 로직을 설명하는 의사코드 주석이 정확하게 작성되어야 합니다.
필수 ④
구두 설명 준비 — 교사가 지정한 코드 부분을 구두로 설명할 수 있어야 합니다. 본인이 작성한 코드를 완전히 이해해야 합니다.
// 01 · Part 3.1 · 의사코드 주석 예시

의사코드 주석 작성 예시

JavaScript / Python 예시
/* * PSEUDOCODE: * FUNCTION searchStudent(name): * FOR each student IN studentList DO * IF student.name == name THEN * RETURN student * END IF * END FOR * RETURN null * END FUNCTION */ function searchStudent(name) { for (let student of studentList) { if (student.name === name) { return student; } } return null; }
주석 포함 기준
✓ 포함 필수
각 함수/기능 블록마다 의사코드 형식의 주석. INPUT/OUTPUT, 조건, 반복 구조 포함.
✗ 불충분
"이 함수는 학생을 검색합니다"처럼 단순 설명만 있는 주석. 의사코드 구조 없음.
⚡ 팁
주요 알고리즘(정렬, 탐색, 데이터 처리)이 포함된 영역을 우선 주석 처리.

Part 3.2:
개발 보고서

5가지 항목으로 구성된 개발 과정 보고서를 작성합니다. (20점)

① 개요 · 2점
프로젝트 요약
웹사이트의 목적, 주요 기능, 구현 결과를 간결하고 명확하게 요약합니다.
② 계획 · 3점
문제 정의 & 초기 설계
해결하려는 문제를 명확히 정의하고, 초기 해결 방향과 접근 방법을 논리적으로 서술합니다.
③ 프로그래밍 설계 · 6점
알고리즘 · 자료구조 · 코드 설명
사용한 알고리즘과 자료구조를 설명하고, 의사코드와 실제 코드를 제시합니다. 핵심 기능 구현 방식 상세 설명.
④ 오류 및 해결 과정 · 6점
피드백 3회 이상 + 문제 해결
개발 중 발생한 오류와 교사 피드백(최소 3회)을 포함합니다. 오류 원인 분석 및 해결 과정을 논리적으로 서술.
⑤ 결론 및 개선점 · 3점
평가 & 한계 & 발전 방향
프로젝트 성과를 평가하고, 한계와 개선 방향을 논리적으로 서술합니다.
// 02 · Part 3.2 · 프로그래밍 설계

③ 프로그래밍 설계 작성법

가장 배점이 높은 항목 (6점). 4가지 요소를 모두 포함해야 최고점을 받습니다.

포함 필수 요소
4가지 모두 포함
① 알고리즘 — 사용한 알고리즘 이름, 선택 이유, Big O 복잡도
② 자료구조 — 사용한 자료구조(리스트, 딕셔너리 등)와 그 이유
③ 의사코드 — 핵심 기능의 의사코드 표현
④ 코드 설명 — 실제 코드 발췌 + 기능 설명
좋은 예시
구체적 서술 방법
❌ "for 루프를 사용하여 데이터를 처리했습니다."
✓ "학생 목록에서 특정 이름을 찾기 위해 선형 탐색(O(n))을 사용했습니다. 데이터가 정렬되지 않아 이진 탐색 사용이 불가능했기 때문입니다."

핵심: 단순히 코드를 나열하는 것이 아니라, 왜 이 방법을 선택했는지 컴퓨팅 사고의 관점에서 설명해야 합니다.

AI 활용,
어디까지?

수행평가 3은 AI 활용이 부분적으로 허용됩니다. 경계를 명확히 이해하세요.

✅ 허용
코딩 보조 — AI를 이용하여 코드 작성 또는 오류 수정 보조 가능. 사용 시 출처 표기 필수. (예: "ChatGPT를 이용해 이 함수의 오류를 수정했습니다.")
❌ 금지
보고서 작성 — 개발 보고서의 모든 항목(개요·계획·설계·오류해결·결론)은 본인이 직접 작성해야 합니다. AI 생성 텍스트 제출 불가.
🗣️ 필수
구두 설명 — 교사가 지정하는 코드 부분을 구두로 설명할 수 있어야 합니다. AI가 작성했더라도 코드의 작동 원리를 완전히 이해해야 합니다.
⚠️ 경고
AI 활용 은폐 — 사용한 AI 도구를 보고서에 명시하지 않으면 학문적 진정성 위반으로 처리됩니다. 코딩에 AI를 썼다면 반드시 보고서에 기재하세요.
