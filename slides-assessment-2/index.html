// 01 · 평가 범위

출제 범위:
알고리즘

B2.4 과정에서 학습한 알고리즘들이 출제됩니다.

탐색 알고리즘 (Search)
  • 선형 탐색 Linear Search — O(n), 정렬 불필요
  • 이진 탐색 Binary Search — O(log n), 정렬 필수
정렬 알고리즘 (Sort)
  • 버블 정렬 Bubble Sort — O(n²), 인접 비교
  • 선택 정렬 Selection Sort — O(n²), 최솟값 선택
Big O 복잡도 비교
알고리즘 최선 최악
선형 탐색 O(1) O(n)
이진 탐색 O(1) O(log n)
버블 정렬 O(n) O(n²)
선택 정렬 O(n²) O(n²)
// 02 · Part 2.1

Part 2.1:
알고리즘 완성

주어진 알고리즘을 세 가지 표현 방식으로 완성합니다. (15점)

①
순서도 완성 — 빈칸이 있는 순서도를 완성합니다. 올바른 기호 사용: 시작/끝(타원), 처리(직사각형), 결정(마름모), 입출력(평행사변형), 흐름선.
②
의사코드 완성 — 알고리즘의 논리를 의사코드로 작성합니다. 제어 구조(IF, WHILE, FOR), 변수 할당, 비교 연산자를 정확히 사용합니다.
③
Python 코드 완성 — 실행 가능한 Python 코드로 작성합니다. 올바른 문법, 들여쓰기, 논리 구조를 갖추어야 합니다.
④
알고리즘 식별 — 완성한 알고리즘의 명칭, 핵심 원리, 적합한 사용 상황을 간단히 설명합니다.
// 02 · Part 2.1 · 순서도 기호

표준 순서도 기호

시작 / 끝
START / END
타원 (둥근 직사각형). 알고리즘의 시작과 끝을 나타냄.
처리 / 연산
PROCESS
직사각형. 변수 할당, 계산, 교환 등의 처리 작업.
결정 / 조건
마름모. 조건을 판단해 두 방향으로 분기.
입력 / 출력
INPUT / OUTPUT
평행사변형. 사용자 입력 또는 화면 출력.
흐름선
→ ↓ ↑ ←
화살표로 실행 흐름 방향을 표시. 반드시 방향 표시 필수.
연결자
A
원. 순서도가 여러 부분으로 나뉠 때 연결 지점 표시.
// 02 · Part 2.1 · 의사코드

의사코드 작성 규칙

기본 구조
# 변수 할당 x ← 0 array ← [3, 1, 4, 1, 5] # 조건문 IF x > 0 THEN output x ELSE output "음수" END IF
반복문
# WHILE 반복 WHILE i < n DO process(i) i ← i + 1 END WHILE # FOR 반복 FOR i ← 0 TO n-1 DO process(i) END FOR

핵심 원칙: 의사코드는 특정 언어에 종속되지 않지만 논리가 명확해야 합니다. 들여쓰기로 블록 구조를 나타내고, 화살표(←)로 대입을 표시합니다.

// 03 · Part 2.2

Part 2.2:
문제 상황 논술

실제 문제 상황을 분석하고 가장 적합한 알고리즘을 논리적으로 선정합니다. (15점)

① 상황 파악
제시된 문제 상황의 조건을 파악합니다. 데이터 크기? 정렬 여부? 탐색인가 정렬인가? 시간 제약?
② 알고리즘 선정
조건에 맞는 알고리즘을 선정하고 왜 다른 알고리즘보다 적합한지 비교 근거를 제시합니다.
③ 동작 원리 설명
선정한 알고리즘이 이 문제 상황에서 어떻게 작동하는지 구체적으로 설명합니다. 예시 데이터와 함께 단계별로.
④ 글 구조 완성
주장 → 근거 → 설명 → 연결의 4단계 구조로 글을 작성합니다. 논리 흐름이 일관되어야 합니다.

// 03 · Part 2.2 · 선정 가이드

알고리즘 선택 결정 가이드

탐색: 선형 vs 이진
  • 선형 탐색 선택 → 데이터가 정렬되지 않음 / 데이터 양이 적음
  • 이진 탐색 선택 → 데이터가 정렬됨 / 데이터 양이 많음 / 빠른 탐색 필요
언제 탐색 알고리즘?
  • 특정 값을 찾는 문제
  • 목록에서 특정 항목 조회
  • 데이터베이스 레코드 검색
정렬: 버블 vs 선택
  • 버블 정렬 → 거의 정렬된 데이터 / 단순 구현 필요
  • 선택 정렬 → 교환 횟수 최소화 필요 / 메모리 쓰기 비용이 클 때
언제 정렬 알고리즘?
  • 데이터를 순서대로 배열해야 할 때
  • 이진 탐색을 위한 전처리
  • 순위/랭킹 시스템 구현
