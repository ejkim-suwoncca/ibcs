// 01 · Course Overview

Two Levels.
One Redesigned Course.

The 2027 syllabus restructures IB CS into two overarching themes instead of numbered topics. Both levels share the same content — HL goes deeper and requires broader connections across the syllabus.

Theme A + Theme B fully, including OOP, Databases, Machine Learning, and the case study — all now standard for every SL student.

// 02 · Course Themes & Content

Theme A + Theme B

The old numbered topics are replaced by two interlocking themes. Theme A covers how systems work. Theme B covers how to solve problems with them. HL students engage with both at greater depth.

Theme A

Concepts of Computer Science

Understanding how computing systems work — from hardware architecture to networking, databases, and machine learning.

A1
Computer Fundamentals
CPU & GPU architecture, fetch-decode-execute, memory hierarchy, OS, logic gates, binary & hex, compilers/interpreters
A2
Networks & Security
Network topologies, protocols, TCP/IP, cybersecurity, encryption, firewalls, cloud computing
A3
Databases Now Core — All Students
Relational databases, SQL queries, entity-relationship models, normalization, data storage
A4
Machine Learning ★ Brand New 2027
Supervised & unsupervised learning, K-NN, decision trees, neural networks (HL), bias & fairness in AI, ethical implications
Assessed → Paper 1 (including Case Study)
Theme B

Computational Thinking & Programming

Solving problems algorithmically — designing, coding, testing, and evaluating solutions using Java or Python.

B1
Computational Thinking
Decomposition, abstraction, algorithm design, flowcharts, pseudocode, time complexity, trace tables
B2
Programming
Variables, data types, control structures, functions, recursion, exception handling, testing & debugging (Java or Python)
B3
Object-Oriented Programming Now Core — All Students
Classes, objects, constructors, inheritance, polymorphism, encapsulation, UML diagrams
B4
Abstract Data Structures HL Extended
Stacks, queues, linked lists, binary trees, graphs — dynamic structures for complex problem-solving scenarios
Assessed → Paper 2 + Internal Assessment
Pre-Seen Case Study — Now for Both SL & HL

Emerging Technology Case Study

Released before the exam period, the case study focuses on a real-world emerging technology (e.g. AI, blockchain, quantum computing). Students research it in depth and apply Theme A concepts to analyse it. Previously only for HL — now every student studies and is assessed on the case study in Paper 1.

// 03 · Assessment Structure

Two Papers.
One IA. No Options.

The 2027 model has been streamlined. Paper 3 is gone. No more option papers. Both SL and HL sit the same two external papers, with HL content built into those papers at higher demand.

// 04 · Internal Assessment

The Computational
Solution

The IA is now called the Computational Solution. Major update for 2027: no client is required — you choose the problem yourself. The focus is the full computational thinking process from definition to ethical evaluation.

IA At a Glance

Develop a working software product for a problem you are genuinely interested in solving. Document every stage from initial specification through to evaluation. Bring together both Theme A and Theme B in your solution.

HOURS
35
teaching hours allocated
WEIGHT
20%
both SL and HL
5 Assessment Criteria
A — Planning & Problem Definition
B — Design of Solution
C — Development
D — Functionality & Testing (+ Video)
E — Evaluation + Mandatory Ethical Review ★

★ New in 2027: Criterion E requires a mandatory ethical review — connect your solution's design and data use to privacy, bias, and societal impact as explored in Theme A4.

The Process (2027)

01

Choose Your Own Problem

No client required. Pick a problem you genuinely want to solve with a software solution. Confirm it has sufficient complexity to demonstrate your skills.

02

Define & Specify

Articulate the problem clearly. Set measurable success criteria. Explain how you will apply the computational thinking process to solve it.

03

Design the Solution

Produce wireframes, data structure designs, UML class diagrams, flowcharts, and algorithm outlines before writing any code.

04

Develop & Document

Write code in Java or Python. Use OOP, appropriate data structures, and algorithms. Comment meaningfully. Document every key design decision.

05

Test & Record Video

Create a testing plan. Test with normal, extreme, and erroneous inputs. Record a video (max 5 minutes) showing functionality and your testing strategy.

06

Evaluate + Ethical Review

Evaluate against success criteria. Discuss limitations honestly. Write your ethical review — privacy, bias, data security, and social impact of your solution.

// 06 · How to Succeed

Tips for the
Two Years Ahead

The 2027 syllabus rewards computational thinkers who engage with real-world technology, write clean code, and start their IA with genuine curiosity.

01

Commit to One Language Early

Java or Python — choose one and become fluent in it. The exams test algorithmic thinking in code. Practice writing and tracing algorithms from day one.

02

Start the IA in Year 1

35 hours of teaching time sounds like a lot. It isn't. Defining a meaningful problem, designing a solution, and building something non-trivial takes far longer than you expect.

03

Research the Case Study Deeply

It's now in Paper 1 for everyone. When it's released, read everything about it. Connect the technology to every sub-topic in Theme A. This is where easy marks are won or lost.

04

Understand ML Conceptually

Machine learning (A4) is brand new. You don't need to implement it — but you must understand how supervised and unsupervised learning work, what bias means, and why it matters.

05

Master OOP Thoroughly

OOP is now core for everyone in B3, and it appears in Paper 2 and the IA. Classes, inheritance, polymorphism, and UML diagrams must become second nature in Year 1.

06

Think Ethically in Every Topic

Ethics is embedded everywhere — from AI bias in A4 to the mandatory ethical review in your IA. Every new concept: ask yourself what the societal and privacy implications are.

The IB Learner Profile still matters: IB CS 2027 is built for students who are inquirers, thinkers, risk-takers, and principled. The ethical dimension is no longer optional — it is built directly into how you are assessed.

// 07 · Semester Schedule & Assessments

Your First
Semester Roadmap

Three sections. Each one builds directly on the last — from hardware understanding, to algorithmic thinking, to coding a real prototype, to beginning your IA.

Weeks 1–7
Section
01

Computing Fundamentals & AI

A1 · Computer Fundamentals B1 · Computational Thinking

We begin by understanding what hardware modern AI actually runs on — CPU, NPU, HBM memory, bus speeds — then zoom out to the logic that underlies all computing: logic gates, truth tables, and the Von Neumann architecture. We close the section by connecting it all to real-world ethical questions.

📋 Formative Task
AI Infrastructure Portfolio

Research report documenting the hardware stack for modern AI — CPU/NPU architecture, HBM memory, bus speeds, logic gates & truth tables. Includes a Social & Ethical Issues section (A1.12) covering energy consumption and the Digital Divide.

⚡ Summative Assessment 1
The "AI Architect" Written Task

Scenario: A hospital needs to implement a local AI diagnostics model with a limited budget and strict privacy laws.

Task: Identify the necessary hardware, explain data flow using Von Neumann principles, and justify all choices based on technical and ethical constraints.

Weeks 8–14
Section
02

Computational Logic & Python Prototype

B1 · Computational Thinking B2 · Programming

The philosophy here is Dissection Before Coding. Before you touch a keyboard, you must break the problem apart. Decompose it into modules. Use Abstraction to decide which variables actually matter. Only then do you write code.

📋 Formative — Weekly Coding Labs
Lab 1 Calculator — applying Selection (if/elif/else)
Lab 2 Password Validator — applying Iteration (loops)
Lab 3 Simple Database — using Lists & Arrays
⚡ Summative Assessment 2
The Python Prototype

Build a functional prototype that solves a real utility problem — examples include a System Resource Simulator or a Project Management Tool.

Required submission: Your working .py file plus your Dissection Map (Flowchart + Pseudocode) — proving you applied Decomposition and Abstraction before coding.

Weeks 17–18
Section
03

IA Pre-Discovery

Internal Assessment Criterion A

You've completed a technical written analysis and built a working program. You now have the foundation to approach the IA with real confidence. These final weeks are dedicated to finding your problem and beginning Criterion A — the most important part of the IA process.

What you bring in
✓A written analysis of hardware & ethics (Assessment 1)
✓A working Python prototype + Dissection Map (Assessment 2)
✓Practiced Decomposition, Abstraction, OOP foundations
What you start here
→Find a real-world problem worth solving
→Draft your Criterion A — Problem Identification & Success Criteria
→Begin scoping the complexity of your solution

How it all connects: Assessment 1 trains your analytical writing. Assessment 2 trains your computational design and coding. By Week 17, you won't be starting the IA cold — you'll already know how to break down a problem, design a solution, and explain it clearly. That's exactly what Criteria A, B, and C demand.
