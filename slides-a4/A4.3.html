// A4.3.1 · Supervised Learning

Linear Regression — Continuous Predictions

Linear regression predicts a continuous numeric output by finding the best-fit line through training data.

The Regression Line
Equation: y = mx + b

x = independent (predictor) variable
y = dependent (response) variable
m = slope — how much y changes per unit of x
b = intercept — value of y when x = 0

The algorithm finds the m and b that minimise prediction error across all training points.
Goodness of Fit — r²
r² (coefficient of determination) measures how well the regression line explains the variation in the data.

r² = 1.0 → perfect fit (line passes through all points)
r² = 0.0 → the line explains nothing
r² = 0.85 → 85% of variation is explained by the model

Higher r² = better model fit.

Example: Predicting a house price (y) based on its size in m² (x). Slope might be: every extra m² adds $2,500 to price. Intercept: base price of $50,000 for a 0 m² house (not physically meaningful, but mathematically defined).

// A4.3.2 · Classification

K-Nearest Neighbours (K-NN)

K-NN classifies a new data point by looking at its K nearest neighbours in the training data and choosing the most common class among them.

1
Choose K (number of neighbours to consider — a hyperparameter)
2
For a new data point, calculate its distance to all training points (commonly Euclidean distance)
3
Select the K training points with the smallest distances (the "nearest neighbours")
4
The new point is assigned to the majority class among those K neighbours
Advantages
Simple to understand and implement
No training phase — "lazy learner"
Works well with small datasets
Disadvantages
Slow at prediction time (compares all training points)
Sensitive to irrelevant features and scale
Suffers in high dimensions (curse of dimensionality)

Real-world application: Collaborative filtering recommendation systems — "users similar to you also liked X."

// A4.3.2 · Classification

Decision Trees

A decision tree classifies data by repeatedly splitting it on the feature that best separates the classes — forming a tree of if-then rules.

Structure
Root node: the first split (best feature)
Internal nodes: subsequent decision splits
Leaf nodes: final class predictions
Each path from root to leaf = one classification rule
Advantages
Highly interpretable — humans can follow the decisions
No scaling required
Handles both numerical and categorical data
Built-in feature importance (embedded feature selection)
EXAMPLE: MEDICAL DIAGNOSIS
Fever > 38°C?
Yes → Cough present?
  Yes → → Flu
  No → → Infection
No → → Healthy

Real-world application: Medical diagnosis — classify patients based on symptoms into possible conditions. Credit scoring — approve or deny loan based on income, credit history, debt ratio.

// A4.3.3 · Evaluation Metrics

Measuring Model Performance

Accuracy alone is often misleading — especially with imbalanced datasets. Four key metrics give a complete picture.

Metric Formula (conceptual) Meaning When it matters
Accuracy Correct predictions / All predictions Overall fraction of correct results Balanced datasets with equal class distribution
Precision True Positives / (True Positives + False Positives) Of all positive predictions, how many were actually positive? When false alarms are costly (e.g., spam filter: don't label good email as spam)
Recall True Positives / (True Positives + False Negatives) Of all actual positives, how many did we catch? When missing a positive is dangerous (e.g., cancer detection: must find all cases)
F1 Score 2 × (Precision × Recall) / (Precision + Recall) Harmonic mean of precision and recall — balances both When both false positives AND false negatives are important

Key insight: A model that predicts "no disease" for every patient might achieve 99% accuracy on a dataset where 99% of patients are healthy — but it has 0% recall and is useless for diagnosis.

// A4.3.3 · Model Evaluation

Overfitting, Underfitting & Hyperparameters

Overfitting
The model learns the training data too precisely — including its noise and random variation.

High accuracy on training data, poor accuracy on new data.

Analogy: memorizing exam answers without understanding the concept
Underfitting
The model is too simple to capture the patterns in the data.

Poor accuracy on both training and new data.

Analogy: studying only the textbook summary and missing important details
Good Fit
The model generalizes well — learns the real patterns without memorizing noise.

Good accuracy on both training and unseen test data.

Goal of hyperparameter tuning
What is a Hyperparameter?
A setting chosen before training that controls how the model learns. Unlike parameters (learned during training), hyperparameters are set by the developer. Examples: K in K-NN, maximum depth of a decision tree, learning rate in neural networks. Hyperparameter tuning = systematically trying different values to find the best-performing combination.
// A4.3.4 · Unsupervised Learning

Clustering — Grouping Without Labels

Clustering groups data points based on similarity in their features — without any predefined labels. The algorithm discovers structure in the data on its own.

How it Works
Data points are grouped so that:
• Points within a cluster are similar to each other
• Points in different clusters are dissimilar

The algorithm determines the number of clusters (K) or discovers it automatically, depending on the technique used.
K-Means Clustering (Concept)
1. Choose K cluster centroids randomly
2. Assign each point to its nearest centroid
3. Move each centroid to the mean of its assigned points
4. Repeat until centroids stop moving

Result: K groups of data points.
Real-World Applications
Market segmentation: group customers by purchasing behaviour to target marketing
Document clustering: group news articles by topic
Anomaly detection: points far from any cluster = outliers / potential fraud
IB Example
Using purchasing data to segment a customer base into high-spenders, occasional buyers, and bargain hunters — without any prior label of which category each customer belongs to.
// A4.3.5 · Association Rule Mining

Association Rules — Finding Hidden Relationships

Association rule mining finds co-occurrence patterns among items in large datasets — "if X then Y" relationships that appear frequently.

Classic Example: Market Basket Analysis
A supermarket finds: customers who buy bread and butter also frequently buy jam.

Rule: {bread, butter} → {jam} with 75% confidence.

This rule can inform shelf placement, promotions, and cross-selling strategies.
Key Terms
Support: how often the items appear together in the dataset
Confidence: given X was purchased, how often is Y also purchased?
Lift: how much more likely is Y given X compared to Y alone?

IB Example — Crime Analysis: Mining police records reveals that areas with high rates of vandalism also frequently experience theft. This assists law enforcement in predictive policing and resource allocation. Association rule: {vandalism} → {theft}.

Note: Association rule mining is unsupervised — no labels are needed. It discovers patterns directly from co-occurrence frequencies in the data.

// A4.3.6 · Reinforcement Learning

Learning Through Trial and Error

An agent interacts with an environment, taking actions and receiving rewards or penalties — learning a policy that maximises cumulative reward over time.

Key Concepts
Agent: the learner / decision-maker
Environment: the world the agent acts in
State: the current situation of the agent
Action: a choice the agent can make
Reward: positive or negative feedback for an action
Policy: the strategy mapping states → actions
Exploration vs Exploitation
Exploitation: use the best known action (safe, but may miss better options)

Exploration: try new actions to discover potentially better rewards (risky, but learns more)

Effective RL agents must balance both throughout training.
Cumulative Reward
The agent does not just optimise the immediate reward — it optimises the total reward over many steps.

Example: in chess, sacrificing a piece (negative short-term reward) might lead to winning the game (large long-term reward).

Real-world applications: Game-playing AI (AlphaGo, chess engines), robot navigation, self-driving cars, dynamic pricing systems.

// A4.3.7 · Genetic Algorithms

Genetic Algorithms — Evolution as Optimization

Genetic algorithms mimic biological evolution to find optimal or near-optimal solutions to complex problems, especially where the search space is too large for exhaustive search.

1
Population: start with a set of random candidate solutions ("individuals")
2
Fitness Function: evaluate how good each candidate solution is at solving the problem
3
Selection: choose the best-performing candidates as "parents" for the next generation
4
Crossover: combine parts of two parent solutions to create new "offspring" solutions
5
Mutation: randomly alter small parts of offspring solutions to maintain diversity and explore new areas
6
Evaluation & Termination: repeat until a solution meets the required fitness, or after a set number of generations

Classic example — Travelling Salesperson Problem (TSP): Find the shortest route that visits N cities exactly once. Genetic algorithms evolve better and better routes over many generations without checking all possible permutations.

// A4.3.8 · Artificial Neural Networks

Artificial Neural Networks (ANNs)

ANNs are inspired by biological neurons. They process input through connected layers of nodes, learning weights during training to produce accurate outputs.

Single Perceptron
Inputs (x₁, x₂ …): features fed in
Weights (w₁, w₂ …): how much each input matters — adjusted during training
Bias: constant offset, shifts the activation threshold
Activation function: decides whether the neuron "fires" (e.g. ReLU, sigmoid)
Output: the neuron's predicted value
Multi-Layer Perceptron (MLP)
Input layer: receives the raw features
Hidden layers: extract increasingly abstract patterns
Output layer: produces the final prediction

Multiple hidden layers = "deep" neural network

// A4.3.9 · Convolutional Neural Networks

CNNs — Learning from Images

CNNs are designed for data with spatial structure (images, video). They automatically learn to detect features — edges, textures, shapes — through convolutional layers.

Input
Input Layer: the raw image data (e.g. 224×224×3 pixels for an RGB image)
Conv
Convolutional Layers: apply small filters (kernels) that slide across the image to detect local features. Each filter detects a different pattern (vertical edges, curves, etc.)
Activ
Activation Functions: applied after each convolution to introduce non-linearity. ReLU (Rectified Linear Unit) is most common — sets negative values to 0
Pool
Pooling Layers: downsample feature maps — reduce size while retaining the most important features. Max pooling takes the maximum value in each region
FC
Fully Connected Layers: connect every neuron to the next layer — combine detected features to make the final classification
Out
Output Layer: produces class probabilities (e.g. 92% dog, 7% cat, 1% other)

Key hyperparameters that affect CNN performance: number of convolutional layers, kernel (filter) size, stride (how far the filter moves each step), activation function choice, loss function used during training.

// A4.3.10 · Model Selection

Choosing the Right Algorithm

No single ML algorithm is best for all problems. Model selection requires understanding the nature of the data and the problem type.

Factor Influences Choice How?
Output type Continuous output → regression; Categorical output → classification; No labels → clustering
Dataset size Small datasets → K-NN, decision trees work well; Large datasets → neural networks benefit more
Interpretability needs Decision trees are easily explained to non-experts; Neural networks are "black boxes"
Feature types Images → CNN; Tabular data → regression, decision trees; Sequential data → RNN
Training time K-NN has no training but slow prediction; neural networks need long training but fast inference
Data structure Labelled data → supervised; Unlabelled → unsupervised; Agent-environment → reinforcement

Key principle: Different algorithms trained on the same data will produce different results. Model selection and comparison (trying multiple algorithms) is a standard practice in ML development.
