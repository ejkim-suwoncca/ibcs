// Foundation · The Hierarchy

AI → ML → Deep Learning

These three terms are often confused. They are nested concepts — each is a subset of the one above.

Artificial Intelligence (AI)
Machine Learning (ML)
Deep Learning
Artificial Intelligence
Computers performing tasks that require human-like intelligence — reasoning, perception, decision-making. Includes rule-based systems, not just ML.
Machine Learning
A subset of AI — algorithms that learn from data without being explicitly programmed for each task.
Deep Learning
A subset of ML — uses multi-layered neural networks. Excels at images, speech, and language tasks.
// A4.1.1 · Type 1

Supervised Learning

The algorithm is trained on labelled data — examples where the correct output is already known. It learns to predict outputs for new, unseen inputs.

Regression
Output is a continuous number

Example: Predict house price based on size and location
Example: Predict tomorrow's temperature
Classification
Output is a category (discrete label)

Example: Classify email as spam/not-spam
Example: Diagnose image as cancer/healthy
ApplicationSupervised TypeWhy supervised?
Email spam filterClassificationTrained on millions of labelled spam/non-spam emails
House price predictionRegressionTrained on historical sale prices with known values
Medical imaging diagnosisClassificationTrained on images labelled by doctors
// A4.1.1 · Type 2

Unsupervised Learning

The algorithm is trained on unlabelled data — no correct answers are provided. It discovers hidden patterns or groupings on its own.

Clustering
Groups data into clusters based on similarity

Example: Customer segmentation (grouping buyers with similar habits without pre-defined categories)
Example: Detecting social groups on a social network
When to use unsupervised
When there's no existing labelled data

When you want to discover unknown patterns — you don't know in advance what categories exist

Often used for exploration and data analysis

Key difference from supervised: There is no "correct answer" to compare against. The algorithm must find structure in raw data without any guidance.

// A4.1.1 · Type 3

Reinforcement Learning

Learning by trial and error. The algorithm (agent) takes actions in an environment and receives rewards or penalties. Over time, it learns to maximise total reward.

How it works
Agent — the model that learns
Environment — what it interacts with
Action — what the agent does
Reward — feedback: positive or negative
Loop: act → observe reward → update strategy
Real-world Applications
Game AI — AlphaGo (Google DeepMind) learned to play Go better than any human

Robotics — teaching a robot arm to grasp objects

Autonomous vehicles — learning to navigate roads

Analogy: Like training a dog. You don't show it the "correct" behaviour — you reward good actions and ignore or correct bad ones. The dog (agent) learns what gets treats.

// A4.1.1 · Types 4 & 5

Deep & Transfer Learning

Deep Learning
Uses multi-layered neural networks — modelled loosely after the human brain

Automatically learns features at multiple levels of abstraction

Applications: image recognition, voice assistants (Siri, Google), language models (ChatGPT)

Subset of machine learning — NOT separate from it
Transfer Learning
Take a model already trained on a large dataset and fine-tune it for a new (related) task

Requires less data and less training time

Example: Take ImageNet-trained model → fine-tune to detect specific flower species

Like learning Spanish if you already know Italian

Common mistake: Deep learning is a subset of machine learning. It is not a completely separate approach. Transfer learning is not just reusing a model — it involves adapting a model trained on one task for a new (but related) task.

// Summary · ML Types

Five ML Approaches Compared

TypeDataHow it learnsExample application
Supervised Labelled (input + correct output) Learn mapping from inputs to outputs Spam filter, medical diagnosis
Unsupervised Unlabelled (no correct outputs) Find patterns/groups in data Customer segmentation, social groups
Reinforcement Rewards/penalties from environment Trial and error to maximise reward AlphaGo, autonomous vehicles
Deep Learning Large labelled datasets Multi-layer neural networks Image recognition, language models

Exam tip: Be ready to identify which ML type is most appropriate for a given scenario and explain your reasoning. Consider whether the data is labelled, whether the goal is prediction or pattern discovery, and what the real-world constraints are.

// A4.1.2 · Hardware

Hardware for Machine Learning

Different ML tasks require different hardware. The scale and type of computation determines the right tool.

CPU — Central Processing Unit
Few, powerful cores — great at sequential tasks

Suitable for: small models, inference on pre-trained models

Limitation: too slow for training large neural networks

Example: Your laptop's Intel or Apple M chip
GPU — Graphics Processing Unit
Thousands of small cores — excellent parallel processing

ML training involves repeating the same operation on huge amounts of data → perfect fit

Industry standard for training neural networks

Example: NVIDIA RTX series
TPU — Tensor Processing Unit
Designed specifically for neural network matrix operations

Faster and more energy-efficient than GPUs for large-scale training

Created by Google; available via Google Cloud / Colab

Example: Google's Cloud TPUs
NPU — Neural Processing Unit
Integrated chip for AI inference on edge devices (phones, laptops)

Very low power consumption — designed for on-device AI

Edge computing — processing near the data source

Example: Apple Neural Engine, Qualcomm AI chips

Cloud vs edge: Training typically happens in the cloud (GPU/TPU); inference (using the trained model) increasingly happens on edge devices (NPU/ASIC) for speed and privacy.

// A4.4 · Ethics

Ethics in Machine Learning

Machine learning systems make decisions that affect people's lives — jobs, loans, medical treatment, criminal justice. This power creates ethical responsibilities.

Bias
Biased training data produces biased models. ML can amplify existing social inequalities.
Privacy
ML systems collect and infer private information in ways people never expected or consented to.
Transparency
Many ML models are "black boxes" — even designers can't fully explain why they made a decision.
Accountability
When an ML system makes a harmful decision, who is responsible — the developer? The company? The data?
Environmental Impact
Training large models consumes enormous energy. GPT-3 training ≈ 500 tonnes of CO₂.
Misinformation
Generative AI can produce convincing fake text, images, and video — enabling disinformation at scale.
// A4.4.1 · Ethical Issue 1

Bias in Training Data

If the training data over- or under-represents particular groups, the model's predictions will be unfair or inaccurate for those groups.

Real Example: Amazon (2018)
Amazon built an AI recruitment tool trained on 10 years of hiring data. Because most past hires were male, the model penalised CVs that mentioned "women's" (e.g. "women's chess club") and downgraded graduates of all-women colleges. Amazon scrapped the tool.
Real Example: Apple Card (2019)
Entrepreneur David Heinemeier Hansson found Apple Card offered him 20× the credit limit of his wife — despite shared finances and her higher credit score. The algorithm's basis for this decision was never made transparent.

Why it's hard to fix: Bias can exist in how data was collected, which features were chosen, how labels were assigned, and which metrics are used to evaluate success. It's not enough to remove obvious biased columns — the bias may be encoded in other features.

// A4.4.1 · Ethical Issue 2 & 3

Privacy & Consent

Privacy Violations
ML can infer private data from public data — e.g., infer health conditions from purchase history, predict political views from social media likes

Real example: Google DeepMind used 1.6 million UK patient health records to develop a kidney disease app without adequate patient consent, breaching UK privacy law
Consent Issues
Large training datasets regularly include personal data collected without explicit consent

Many companies train ML models on customer databases or sell customer data to data-matching companies

Question: How much control should individuals retain over their personal information?

IB exam tip: When discussing ethical issues, you must present both sides. Privacy protection can conflict with beneficial applications (medical diagnosis, personalised education). The question is where to draw the line.

// A4.4.1 · Ethical Issues 4 & 5

Transparency & Environment

Lack of Transparency
Many ML models — especially deep neural networks — are "black boxes": even developers cannot explain exactly why a specific decision was made

This is a problem when decisions affect people: loan rejections, medical diagnoses, parole decisions

Explainability (XAI) is an active research area trying to make models more interpretable
Environmental Impact
Training a single large language model (LLM) can emit as much CO₂ as 5 cars over their lifetimes

Global data centre power use expected to exceed 1000 TWh by 2026 (double 2022 levels)

GPT-3 training: approximately 500 metric tonnes of CO₂

Balancing AI capability with sustainability is a growing challenge

Societal impact: As ML systems are used to make decisions in education, healthcare, criminal justice, and hiring, their societal impact grows. Designing ethical AI requires diverse teams, inclusive datasets, and ongoing auditing.

// A4.4.2 · Ethics as Technology Evolves

Reassessing Ethics Over Time

As technology evolves, new ethical questions emerge that weren't anticipated when systems were first built.

①
Facial Recognition & Surveillance — governments and corporations using public cameras with real-time facial recognition. Who is watched? Who decides when to watch?
②
Deepfakes & Misinformation — AI-generated fake videos and images make it increasingly impossible to verify authentic information. Elections and public opinion become vulnerable.
③
Autonomous Decision-Making — who is responsible when an autonomous vehicle injures someone? Or an AI recommends the wrong medical treatment?
④
Job Displacement — automation driven by ML replaces human jobs. How should societies respond? Retraining? Universal basic income? Who bears responsibility?
⑤
Quantum Computing Threat — future quantum computers may break current encryption, exposing all previously secured communications. Quantum-resistant cryptography is urgently needed.

Key insight: Ethical frameworks written today may be outdated tomorrow. Society needs ongoing, participatory conversations — not one-time regulations.

// Exam Strategy · Ethics

Answering Ethics Questions

Q
"Discuss the ethical implications of a city using facial recognition AI on its public CCTV network." [4 marks]
✓
Good structure: State the concern → Explain how it arises → Give a real or plausible example → Acknowledge the counterargument → Reach a conclusion. Balance at least 2 pros and 2 cons for a "Discuss" question.
For (benefits)
✓ Reduces violent crime in public spaces
✓ Speeds up identification of missing persons
✓ Can identify known terrorists or offenders
✓ Deters criminals — visible surveillance effect
Against (concerns)
✗ Mass surveillance violates privacy rights
✗ Biased models misidentify minorities at higher rates
✗ Chilling effect — people self-censor in public
✗ No consent given by individuals being scanned
