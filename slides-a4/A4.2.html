// Motivation · Why Preprocessing?

Garbage In, Garbage Out

Machine learning models learn patterns from data. If the data is noisy, incomplete, or biased — the model will learn those flaws, not the underlying truth.

Raw Data Problems
Missing values (null / NaN)
Duplicate records
Outliers and anomalies
Wrong formats / typos
Irrelevant features (noise)
Preprocessing Steps
1. Data Cleaning — fix / remove bad data
2. Feature Selection — keep only useful columns
3. Dimensionality Reduction — compress features
4. Normalization — scale values consistently
Result
Higher model accuracy
Faster training
Better generalization to new data
Reduced overfitting
More interpretable results

Rule of thumb: Data scientists spend up to 80% of their time on data preprocessing — it is the most critical phase of any ML pipeline.

// A4.2.1 · Data Cleaning

Impact of Data Quality on Model Performance

The training data is the model's entire "world view." Poor quality data leads directly to poor predictions.

Low Quality Data → Poor Model
Missing values cause incorrect averages and biased predictions

Outliers pull regression lines away from the true trend

Duplicates make the model over-weight certain patterns

Inconsistent formats (e.g. "01/05" vs "May 1st") break feature parsing
High Quality Data → Better Model
Complete data ensures all patterns are represented

Consistent values allow reliable comparisons across rows

Accurate labels (for supervised learning) ensure correct targets

Representative data reduces bias in the trained model

Real example: A medical diagnosis model trained on patient records with 30% missing test results will systematically misclassify patients whose results are absent — regardless of how sophisticated the algorithm is.

// A4.2.1 · Data Cleaning Techniques

Fixing Dirty Data

Problem Technique Example
Outliers Remove if data entry error; cap/transform if genuine extreme value; use robust algorithms A patient's age recorded as 999 → clearly an error; remove the row
Duplicate records Remove or consolidate duplicates based on unique identifiers Same customer appearing twice in a sales database with slightly different spellings
Incorrect / inconsistent data Validate against rules; standardize formats (dates, units, categories) Height recorded in both cm and inches → convert all to cm
Irrelevant features Filter out columns that add noise but no predictive signal Removing a customer's "ID number" from a churn prediction model
Improperly formatted data Parse and transform strings, dates, categories into correct types "True" / "False" strings → boolean 1 / 0
Missing data — Imputation Replace missing values with mean, median, or mode; or use a predictive model to estimate the value Missing age → replace with the median age of all patients
Missing data — Deletion Remove rows or columns with too many missing values (if data is plentiful) A column with 70% missing values contributes little — drop it

Imputation vs Deletion: Imputation preserves data volume (important with small datasets) but may introduce bias. Deletion is simpler but wastes data. The choice depends on how much data is missing and why.

// A4.2.1 · Scaling

Normalization vs Standardization

Features with very different numeric ranges can cause ML algorithms to unfairly weight larger-valued features. Scaling puts everything on the same level.

Normalization (Min-Max Scaling)
Rescales values to a fixed range — usually [0, 1]

Formula: x' = (x − xmin) / (xmax − xmin)

Best for: when the distribution is unknown or not Gaussian; image pixel values (0–255 → 0–1)

Risk: sensitive to outliers — one extreme value compresses all others
Standardization (Z-Score Scaling)
Rescales so that values have mean = 0 and standard deviation = 1

Formula: x' = (x − μ) / σ

Best for: when data follows a roughly Gaussian (bell curve) distribution; algorithms like SVM, linear regression

Advantage: less sensitive to outliers than min-max normalization

Example: A dataset has age (20–80) and salary (30,000–200,000). Without scaling, a K-NN classifier will treat salary differences as far more important than age differences — purely because of the numeric scale.

// A4.2.2 · Feature Selection

Choosing the Right Features

A feature is any measurable attribute used as input to the model. Not all features are equally useful — some add noise, some are redundant, some are irrelevant.

What is Feature Selection?
The process of identifying and retaining only the most informative attributes from the dataset before training.

Reduces model complexity, speeds up training, and often improves accuracy by removing misleading signals.
Why Too Many Features is Bad
Overfitting: model memorizes noise alongside real patterns

Slow training: more features = exponentially more computation

Interpretability: harder to understand a model with 500 inputs vs 10

Example: Predicting house prices. Features might include: square footage, number of bedrooms, postcode, owner's favourite colour. Square footage is highly relevant; owner's favourite colour is not — it should be removed.

// A4.2.2 · Feature Selection Strategies

Three Strategies Compared

Filter
Filter Methods — evaluate each feature independently of the learning algorithm, using statistical measures (e.g. correlation, chi-squared test).
✅ Fast and computationally cheap   ❌ Ignores interactions between features
Example: Remove any feature with correlation < 0.1 to the target variable.
Wrapper
Wrapper Methods — use the actual ML model to evaluate subsets of features. Try combinations, keep the best-performing subset.
✅ Considers feature interactions   ❌ Very slow for large feature sets
Example: Forward selection — start with 0 features, add one at a time, keep if accuracy improves.
Embedded
Embedded Methods — feature selection is built into the model training process itself.
✅ Efficient — no separate selection step   ✅ Considers interactions
Example: Decision trees rank features by importance during training; low-importance branches are pruned.

IB Focus: Know the name of each method, roughly how it works, and one advantage and disadvantage for each.

// A4.2.3 · Dimensionality Reduction

The Curse of Dimensionality

A "dimension" = one feature (one column in the dataset). As the number of features grows, data becomes increasingly sparse — and models struggle to find meaningful patterns.

Overfitting
With many features, the model can fit the training data perfectly by memorizing noise — but performs poorly on new data.
Computational Complexity
More features = more calculations per training step. Training time can grow exponentially with dimensionality.
Data Sparsity
In high-dimensional space, data points are far apart. Distance-based algorithms (like K-NN) lose effectiveness when all points seem equally distant.
Memory Usage
High-dimensional datasets consume significantly more RAM and storage — limiting what hardware can process.
Data Visualization
Humans can visualize 2D or 3D space. Beyond 3 dimensions, we cannot directly inspect or verify patterns in the data.
// A4.2.3 · Dimensionality Reduction

Reducing Dimensions Without Losing Information

Dimensionality reduction compresses the dataset from many features into fewer, while preserving the most important variation in the data.

Feature Selection (vs) Feature Extraction
Feature Selection — choose a subset of existing features and discard the rest

Feature Extraction — create new combined features that capture the same information in fewer dimensions (e.g., PCA — note: PCA/LDA details are beyond IB scope)

Both reduce the number of input dimensions.
What Gets Preserved
The goal is to retain the most informative aspects of the data — the patterns and variations that allow the model to distinguish between classes or predict outcomes.

Some information loss is acceptable if the remaining features are still representative.

Sample size consideration: As dimensions increase, exponentially more training samples are needed to maintain the same statistical confidence. Reducing dimensions lowers this requirement.

IB Note: You need to describe why dimensionality reduction is important (curse of dimensionality, overfitting, speed) — not the mathematical details of PCA or LDA.

// Putting It All Together

The ML Preprocessing Pipeline

Preprocessing steps are not isolated — they form a sequential pipeline applied before model training.

Raw Data
→
Data Cleaning
→
Feature Selection
→
Dim. Reduction
→
Normalization
→
Model Training
Order Matters
Clean before selecting — you don't want outliers to distort correlation scores.
Select features before reducing dimensions — dimensionality reduction is most effective on relevant features.
Normalize after selection — to ensure consistent scales.
Train / Test Split
Preprocessing should be fitted on training data only — then applied to test data.

Example: compute the mean for imputation using only training rows; apply that same mean to fill gaps in test rows. This prevents data leakage.
// Worked Example

Case Study: Hospital Patient Data

Goal: predict whether a patient will be readmitted within 30 days.

1
Raw problems found: 15% of blood pressure readings missing; patient ID numbers present; age recorded as "999" for 3 rows; some records duplicated from system migration.
2
Data Cleaning: Remove rows with age = 999 (clearly erroneous). Drop exact duplicate records. Impute missing blood pressure values with the median for patients of the same age group.
3
Feature Selection (Filter): Compute correlation of each feature with the readmission label. Drop "patient ID" (no predictive value). Drop "hospital name" (irrelevant for clinical prediction). Retain age, diagnosis codes, number of prior visits, blood pressure.
4
Normalization: Age (0–120) and prior visits (0–30) are on different scales. Apply min-max normalization to bring both to [0, 1] range so neither dominates the model.
5
Result: Clean, consistently scaled, 8-feature dataset ready for training — instead of the original 45-feature, noisy, incomplete raw file.

Key Concepts Summary

Data Cleaning (4.2.1)
Outliers → remove or cap
Duplicates → remove/consolidate
Missing data → imputation or deletion
Normalization → min-max [0,1]
Standardization → mean=0, σ=1
Bad formats → parse & transform
Feature Selection (4.2.2)
Identify most informative features
Filter: statistical measure (fast, no interactions)
Wrapper: model-based subsets (slow, thorough)
Embedded: built into training (efficient)
Reduces overfitting + training time
Dimensionality Reduction (4.2.3)
Curse of dimensionality concerns:
• Overfitting
• Computation complexity
• Data sparsity
• Memory usage
• Distance metric ineffectiveness
PCA/LDA: beyond IB scope

Common Exam Mistake: Confusing normalization (rescale to [0,1]) and standardization (rescale to mean=0, σ=1). Know both formulas conceptually and when to use each.
