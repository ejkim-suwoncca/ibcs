// The Boot Sequence — Revealed

Order from Chaos

The real sequence. Compare it to what you guessed.

01
Power Button
PSU activates the system.
→
02
BIOS / UEFI
CPU reads firmware from ROM chip.
→
03
POST
Hardware self-check. Verifies all components.
→
04
Bootloader
Found on storage (SSD/HDD). Hands control to OS.
→
05
OS → RAM
Operating system loaded into RAM. Ready to run.

Each step depends on the one before it. Remove any one piece — the system halts. This is what makes it a system.

Activity 02 Acting Out the Boot Sequence 15 min

You are the hardware.

Each of you gets a role. We'll run the boot sequence as a class — step by step.

⚡ Power Supply
You wake up first. Send the signal to the motherboard. Without you, nothing moves.
🧠 CPU
You run the BIOS. Then you run the POST. Then you load the OS. You are always in charge.
💾 Storage (SSD/HDD)
You hold the OS. Wait for the CPU to call on you, then hand over the bootloader.
⚡ RAM
You receive the OS once it's loaded. You are empty at first — volatile, waiting.
🔧 BIOS / UEFI
You run the POST. Check that all the other roles are present. Announce: "All clear."
🖥️ Operating System
You go last. Only appear once RAM is ready. You are what the user sees.
Activity 03 Desktop Disassembly 10 min

Open it up. Find the parts.

Safety first: Unplug the computer. Handle components carefully. Keep screws organized.

🔌
Motherboard
The main circuit board. Everything connects here. Find the CPU socket, RAM slots, PCIe slots.
🧠
CPU
Under the heatsink/fan. Do not remove. Locate the model label on the heatsink.
⚡
RAM
Long sticks in the slots. Read the tiny label — note capacity (GB) and type (DDR3/4).
💾
Storage
HDD (spinning) or SSD. Find the capacity label. Trace the cable to the motherboard.
⚡
Power Supply
The metal box with the fan. Find the wattage rating on the label. Count how many cables come out.
🎮
GPU (if present)
Plugged into a PCIe slot. Not all desktops have a dedicated GPU — some use the CPU's built-in graphics.
Activity 04 Component Research — Jigsaw 10 min

Your group owns one component.

Use your textbook. Answer these three questions about your assigned component.

Q1
What does this component do? Describe its core function in one or two sentences.
Q2
What role does it play during startup? When in the boot sequence does it become active?
Q3
Why is it necessary? What would fail if this component were missing or broken?

Assigned components: CPU · RAM · Storage · Motherboard · Power Supply · GPU

Activity 05 Peer Teaching — Jigsaw Share 10 min

Now teach it to someone else.

If you can explain it clearly, you understand it. If you can't, you know what to review.

Explain the Function
What does your component do? No jargon — explain it like your partner has never seen a computer before.
Show Its Location
Point to it on the open desktop. "This is it. This is where it lives. This is what it connects to."
Its Role in Boot
When does your component activate? Before POST? After? Does it receive data or send it?

Listening: As others present, write down one thing you didn't know about their component.

Activity 06 Systems Diagram Creation 10 min

Draw the whole system.

Paper, pencil, arrows. Your diagram must show all three layers.

Layer 1 — Hardware
Draw and label:
· Power Supply
· Motherboard
· Storage (SSD/HDD)
· CPU
· RAM
Layer 2 — Boot Sequence
Add numbered arrows showing the order data flows between components during startup.
Layer 3 — CPU Micro-Level
Zoom into the CPU and show:
· ALU
· Control Unit
· Registers (extension)

Label what each part does.

Arrows matter: Each arrow should show the direction data travels, not just that two components are connected.
