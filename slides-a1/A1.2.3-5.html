// A1.2.3 · Logic Gates

What are Logic Gates?

Logic gates are the fundamental building blocks of digital circuits. They take binary inputs and produce a binary output based on a logical rule.

Made of Transistors
Each gate is built from transistors acting as electronic switches — allowing or blocking current flow.
Binary In / Out
Inputs and outputs are always 0 (low / off) or 1 (high / on). No in-between.
Two Categories
Basic: AND, OR, NOT
Derived: NAND, NOR, XOR, XNOR

// A1.2.4 · Truth Tables

Building a Truth Table

A truth table lists every possible input combination and its output. For n inputs, there are always 2ⁿ rows.

01
Count inputs → calculate rows: 1 input = 2 rows, 2 inputs = 4 rows, 3 inputs = 8 rows
02
Fill the rightmost input column alternating 0, 1, 0, 1…
03
Fill the next column alternating every 2 rows: 0, 0, 1, 1, 0, 0, 1, 1…
04
Fill the leftmost column alternating every 4 rows: 0, 0, 0, 0, 1, 1, 1, 1…
05
Add intermediate columns (P, Q, R…) for each sub-result, then fill the final output column X

Key point: Every possible combination of 0s and 1s must appear exactly once.

// A1.2.4 · K-Maps

Karnaugh Maps — Simplifying Expressions

K-maps let you visually simplify Boolean expressions — reducing the number of gates needed.

The Big Idea
Plot the 1s from your truth table onto a grid. Group adjacent 1s in powers of 2 (1, 2, 4, 8…). Each group simplifies to a shorter expression.
Key Rules
• Groups must be powers of 2 in size
• Groups must be rectangular / square
• Groups can wrap around edges
• Groups can overlap
• Aim for the fewest, largest groups
• Use Gray code for column headings (00, 01, 11, 10)
Reading a Group
Look at each variable across the group:
• If the variable stays the same → keep it
• If the variable changes → discard it

Then combine groups with OR ( + )

Example: K-map of A·B·C + A·B·C̄ + Ā·B·C simplifies to B·C + A·C

// A1.2.5 · Logic Diagrams

Drawing Logic Diagrams

Convert a Boolean expression into a circuit diagram step by step.

01
Identify inputs — draw a line for each variable (A, B, C…)
02
Work inside brackets first — add gates for innermost expressions
03
Apply NOT gates to inverted variables (Ā, B̄…) before connecting them
04
Connect sub-expressions using AND / OR / XOR gates as indicated
05
Final gate produces output X or Y
Example: Y = (A · B) + (Ā · B̄)
1. AND gate: A, B → output P
2. NOT gate on A → Ā
3. NOT gate on B → B̄
4. AND gate: Ā, B̄ → output Q
5. OR gate: P, Q → output Y

Tip: Simplify with a K-map first — fewer gates = a cleaner, more efficient diagram.
