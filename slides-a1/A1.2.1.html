// A1.2.1 · Motivation

Why Binary?

⚡ Matches Electrical Signals
Transistors have two states:

ON (1) = high voltage (~3–5V)
OFF (0) = low voltage (~0V)

Binary maps perfectly to physics.
🛡️ Robust in Noisy Environments
Only 2 states to distinguish — even with electrical interference, it's easy to tell 0 from 1. More states = more room for error.
🧮 Represents Everything
Binary can encode:

  TRUE / FALSE
  YES / NO
  Numbers
  Text (ASCII / Unicode)
  Images, Sound, Video

👉 With enough bits, any information can be represented as 0s and 1s.

// Base 10 · Denary

Base 10 — Denary

Each column is a power of 10. The digit tells you how many of that power you have.

10⁴ 10³ 10² 10¹ 10⁰
10,000 1,000 100 10 1
0 0 1 0 0

1 × 100  +  0 × 10  +  0 × 1  =  100

Example:  1024

10³10²10¹10⁰
1000100101
1024

1×1000 + 0×100 + 2×10 + 4×1 = 1024  👉 Each column = power of 10

// Base 2 · Binary

Base 2 — Binary

Each column is a power of 2. Each digit can only be 0 or 1.

Power 2³2²2¹2⁰
Value 8421
Bit 1010

Bits that are 1:  8 + 2 = 10 (in decimal)

👉 1010₂ = 10₁₀  ·  Each column = power of 2  ·  Only add values where bit = 1

// 1 Byte = 8 Bits

A Byte — 8 Bit Positions

Each position has a fixed value. The bit (0 or 1) tells you whether to include it.

Position Bit 7Bit 6Bit 5Bit 4 Bit 3Bit 2Bit 1Bit 0
Power 2⁷2⁶2⁵2⁴ 2³2²2¹2⁰
Value 128643216 8421
Bit 0000 0000

👉 1 byte = 8 bits  ·  Positions run from 2⁷ (MSB) → 2⁰ (LSB)

MSB = Most Significant Bit (leftmost, highest value)
LSB = Least Significant Bit (rightmost, value = 1)

// Maximum Value of 1 Byte

All bits = 1

When every bit position is turned ON, you get the maximum value a single byte can hold.

Value 128643216 8421
Bit 1111 1111
128 + 64 + 32 + 16 + 8 + 4 + 2 + 1 = 255

1 byte stores values from 0 to 255 = 256 possible values (2⁸ = 256)

All zeros = 0 (minimum)  ·  All ones = 255 (maximum)

// Terminology

Bits vs. Bytes

Term Size Meaning Range

Kilobyte 1,024 B 1,024 bytes ≈ 1,000 B
Megabyte 1,024 KB 1,024 kilobytes ≈ 1,000,000 B

👉 1 byte = values from 0–255. All text, numbers, and colours in a computer are stored using bytes.

Binary → Decimal
Positional Method

Write the bit under each column value. Add only the columns with a 1.

1010₂  =  8 + 2  =  10₁₀

MSB = Most Significant Bit (leftmost). Start there and work right.

03

// Conversion Method 3

Decimal → Binary
Division Method

LSB = Least Significant Bit (rightmost). Divide by 2, record remainders, read bottom to top.

01
Divide the number by 2
02
Record the remainder (0 or 1)
03
Use the quotient as the new number
04
Repeat until quotient = 0
05
Read remainders from BOTTOM to TOP → that's your binary

EXAMPLE: 10 → binary

DivideQuotientRemainderBit
10 ÷ 250LSB
5 ÷ 221
2 ÷ 210
1 ÷ 201MSB ↑

Read bottom to top:  1 0 1 0  ✓

// Conversion Method 4

Decimal → Binary
Subtraction Method

Use the powers of 2 table. Work left to right — subtract the largest power that fits.

Value 1286432168421
Bit 0000 1010
✓
10 ≥ 8  →  write 1,   10 − 8 = 2 remaining
✗
2 < 4  →  write 0,   move on
✓
2 ≥ 2  →  write 1,   2 − 2 = 0 remaining
✗
0 < 1  →  write 0

Result:  1010₂  =  10₁₀  ✓

// Base 16

Hexadecimal — Base 16

Hex uses 16 symbols. When we run out of digits after 9, we use letters.

Decimal01234567 89101112131415
Hex 0123 4567 89 A B C D E F
Binary 0000000100100011 0100010101100111 1000100110101011 1100110111101111

👉 Each hex digit = exactly 4 bits  ·  0–9 then A–F

A=10, B=11, C=12, D=13, E=14, F=15 — letters continue after 9

// Why We Use Hex

Why Hexadecimal?

Binary is exact but hard to read. Hex is a compact shorthand.

Binary — Precise but long
1101 1010
8 characters. Hard to read, easy to mistype.
Hex — Compact shorthand
D A
2 characters. Same value — much easier to read.

👉 Shorter and easier to read — used in memory addresses, colour codes, error codes, MAC addresses

// IB Requirement · Binary ↔ Hex

Binary ↔ Hex Conversion

Group binary into blocks of 4 bits. Each group = exactly one hex digit.

Left group Right group
Value 8421 8421
Bit 1101 1010
Sum 8+4+1 = 13 8+2 = 10
Hex D A

1101 1010₂  =  DA₁₆

👉 IB requirement: Binary ↔ Hex via 4-bit grouping. Always pad to full groups of 4.
