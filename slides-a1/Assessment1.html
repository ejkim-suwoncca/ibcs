// 01 · CPU

CPU — The Brain

The CPU runs every instruction. It repeats one loop billions of times per second — Fetch → Decode → Execute.

① Fetch
Gets the next instruction from RAM.
RAM에서 다음 명령어를 가져온다.
② Decode
The Control Unit figures out what to do.
제어 장치가 무엇을 할지 해석한다.
③ Execute
The ALU performs the calculation.
ALU가 연산을 수행한다.
Control Unit (CU)
Manages all other parts — does not compute itself.
모든 부품을 관리. 직접 연산하지는 않음.
Cache
Ultra-fast memory inside the CPU chip. CPU checks cache before reaching RAM — much faster.
CPU 칩 내부의 초고속 메모리. RAM보다 훨씬 빠르다.

AI context: Opening an AI app = the CPU fetching and executing its instructions, one by one, millions of times per second.

AI 앱 실행 = CPU가 명령어를 하나씩 초당 수백만 번 실행하는 것.

// 02 · Binary & The Bus

Binary & The Bus — 0s and 1s

Computers only understand two states: 1 (electricity on) and 0 (electricity off). Everything — text, images, instructions — is just a long sequence of 0s and 1s.

컴퓨터는 두 가지 상태만 이해한다: 1(전기 켜짐)과 0(전기 꺼짐). 모든 것은 0과 1의 나열이다.

8 bits = 1 byte = one letter, one colour, or part of an instruction / 문자 하나, 색상 하나, 또는 명령어의 일부

Instructions are binary
When you write code or click a button, it eventually becomes a pattern of 0s and 1s. The CPU reads that pattern and acts on it.
코드나 버튼 클릭은 결국 0과 1의 패턴이 된다. CPU가 이를 읽고 행동한다.
The Bus — Electrical Cables
The bus is simply the wires inside the computer that carry these 0/1 signals between the CPU, RAM, GPU, and Storage.
버스는 CPU, RAM, GPU, 저장장치 사이에서 0/1 신호를 전달하는 전기 케이블이다.
// 03 · GPU

GPU — Pixels & AI

The GPU has thousands of tiny cores that all run at the same time — perfect for tasks that need millions of small calculations done at once.

GPU는 수천 개의 소형 코어가 동시에 작동한다 — 수백만 개의 작은 연산을 한꺼번에 처리하는 데 최적화됨.

Gaming — Rendering Pixels
A 1080p screen = 2+ million pixels, updated up to 144× per second. The GPU calculates every pixel's colour on each frame — simultaneously.
1080p = 200만 픽셀, 초당 최대 144번 갱신. GPU가 각 픽셀의 색상을 동시에 계산.
AI — Parallel Calculations
Training AI = billions of tiny maths operations at the same moment. GPUs handle this perfectly. ChatGPT was trained on thousands of NVIDIA A100 GPUs.
AI 훈련 = 수십억 개의 연산을 동시에. ChatGPT는 수천 개의 NVIDIA A100 GPU로 훈련됨.

GPU — updating pixels every frame

// 04 · RAM

RAM — Working Memory

RAM holds what the computer is using right now. It is fast — but it forgets everything when the power goes off.

Volatile VOLATILE
No power = all data gone instantly.
전원 없음 = 모든 데이터 즉시 사라짐.
Fast FAST
The CPU reads RAM in nanoseconds. Programs must load into RAM to run.
CPU는 나노초 단위로 RAM을 읽는다. 프로그램은 실행 전에 RAM에 로드된다.
❌ Not Saved / ✓ Saved
While you type, your work is only in RAM. Pressing Save copies it to Storage. Power off before saving = gone.
입력 중 작업은 RAM에만 있다. 저장하면 저장장치로 복사된다. 저장 전 전원 차단 = 사라짐.

Why isn't the OS in RAM? RAM is wiped on every shutdown. Windows and macOS must survive restarts — so they live on the SSD and are loaded into RAM at boot.

운영체제는 재부팅 후에도 살아남아야 한다. 그래서 SSD에 저장되고, 부팅 시 RAM으로 로드된다.

// 05 · Storage

Storage — Long-Term Memory

Non-volatile — keeps data even with no power. Your OS, documents, and photos all live here permanently.

비휘발성 — 전원 없이도 데이터를 유지한다. 운영체제, 문서, 사진이 여기에 영구 저장된다.

HDD — Hard Disk Drive SLOW
Spinning magnetic disk + moving arm. Physical movement = slower and fragile.
~100 MB/s

NVMe SSD FAST
Flash memory chips — no moving parts. Reads data electronically.
~3,500–10,000 MB/s
플래시 메모리 칩, 움직이는 부품 없음 → 훨씬 빠르고 내구성이 높다.

// 06 · RAM vs Storage

The Key Difference

Think of it as a desk and a filing cabinet.

RAM · Your Desk
🗂
Fast. Only holds what you're working on right now.
Leave the office (power off) — desk is cleared.
VOLATILE
빠름. 지금 작업 중인 것만 보관. 전원 끄면 지워짐.
Storage · Filing Cabinet
🗄
Slower. Keeps everything permanently.
Files stay even when the office is closed.
NON-VOLATILE
느림. 모든 것을 영구 보관. 전원 꺼도 파일 유지.

Press Save → work moves from RAM → Storage.
Reboot → OS loads from Storage → RAM.

저장 = RAM → 저장장치.  재부팅 = 저장장치 → RAM으로 OS 로드.

// 07 · Compression

Compression — Why Files Shrink

Compression encodes repeated or redundant data more efficiently. Without it, YouTube streaming would be impossible.

압축은 반복되거나 중복된 데이터를 효율적으로 인코딩한다. 압축 없이는 유튜브 스트리밍이 불가능하다.

Without Compression
1 minute of raw 1080p video ≈ ~10 GB.
Impossible to upload or stream.
1분짜리 원본 1080p 영상 ≈ 약 10GB. 업로드·스트리밍 불가능.
With Compression (H.264)
Same 1-minute video ≈ ~150 MB — 60× smaller.
Stores and streams easily.
압축 후 동일 영상 ≈ 약 150MB — 60배 작아짐. 저장·스트리밍 가능.
UncompressedCompressed
Storage / 저장 공간~10 GB / min~150 MB / min
Bandwidth / 대역폭~1.4 Gbps~5 Mbps
Streaming / 스트리밍NoYes

How? A blue sky has thousands of identical blue pixels. Instead of storing each one, compression stores "5,000 blue" as a single instruction — reducing file size drastically.

파란 하늘 = 수천 개의 같은 픽셀. "파란색 5,000개"를 하나의 명령으로 저장 → 파일 크기 대폭 감소.

// 08 · The Big Picture

How It All Connects

Opening a game — all five parts working together.

1
Storage (SSD) — Game files stored permanently here. Press Play → SSD reads the data.
게임 파일이 SSD에 영구 저장. Play 누름 → SSD가 데이터를 읽음.
2
RAM — Game data loaded into RAM so the CPU can reach it fast.
게임 데이터가 RAM에 로드됨 → CPU가 빠르게 접근.
3
Bus — Electrical cables carry 0s and 1s between CPU, RAM, and GPU.
전기 케이블이 CPU, RAM, GPU 사이에 0과 1 신호를 전달.
4
CPU — Runs game logic (movement, physics, AI enemies) via Fetch-Execute.
CPU가 게임 로직(이동, 물리, 적 AI)을 실행.
5
GPU — Receives scene data, calculates every pixel's colour, draws the frame — up to 144× per second.
장면 데이터 수신 → 모든 픽셀 색상 계산 → 초당 최대 144번 화면 출력.

Remove any part and it breaks. No RAM = can't load. No GPU = no image. No Bus = nothing communicates. No Storage = nothing to load from.

어느 부품을 제거해도 작동하지 않는다. RAM 없음 = 로드 불가. GPU 없음 = 화면 없음. 버스 없음 = 통신 불가. 저장장치 없음 = 로드할 것 없음.

Key Terms Glossary

Everything you need for the questions.

Term / 용어Definition / 정의

Byte8 bits. Can represent 256 different values — a letter, colour, or number. / 8비트. 문자, 색상, 숫자 등 256가지 값 표현 가능.

CacheUltra-fast memory inside the CPU chip — faster than RAM. / CPU 칩 내부의 초고속 메모리 — RAM보다 빠르다.
