// Slide 2 · The Processor

CPU — Central
Processing Unit

What it does
· Main processor of the computer
· Executes instructions from programs
· Controls system operations — coordinates every component
Control Unit (CU)
Directs all operations. Tells other parts what to do and when.
ALU
Performs all arithmetic and logical operations (+, −, AND, OR…).

Think of it as: CU = the manager. ALU = the calculator. Together they process everything.

Registers — Fast
CPU Storage

RegisterFull NameRole
PCProgram CounterHolds the address of the next instruction to fetch
MARMemory Address RegisterHolds the memory address being accessed

👉 Registers are used during the Fetch–Decode–Execute cycle. They are the fastest storage in the entire system.

// Slide 4 · Data Transfer

Buses — How Components
Communicate

📦 Data Bus
Transfers the actual data between components (CPU ↔ RAM ↔ Storage). Bidirectional.
📍 Address Bus
Carries memory addresses — tells the system where data is located. Unidirectional (CPU → memory).
🎛️ Control Bus
Carries control signals — read/write commands, interrupts, clock signals. Manages timing.

👉 Buses allow the CPU, memory, and storage to communicate. Without buses, components are isolated islands.

// Slide 5 · CPU Performance

Cores — Processing Power

What is a Core?
A core is an independent processing unit inside the CPU. Each core can handle its own tasks simultaneously.
Single-Core
One task at a time. Tasks must wait in a queue. Older CPUs.
Multi-Core
Multiple simultaneous tasks. Dual (2), Quad (4), Octa (8), and beyond.

👉 More cores = better multitasking. AI training, video editing, and gaming all benefit enormously from more cores.

// Slide 6 · Graphics Processing

GPU — Graphics Processing
Unit

What it does
· Processes graphics, images, and video
· Designed for parallel processing — thousands of small cores
· Originally for graphics; now critical for AI training
CPU
Few powerful cores (4–16). Great at complex, sequential tasks. General purpose.
GPU
Thousands of small cores (1000s). Great at many simple tasks at once. Specialized.

AI connection: Training neural networks requires billions of parallel multiplications — exactly what a GPU is built for.

// Slide 7 · Primary Memory

RAM — Random Access
Memory

What RAM is
· Stores data currently in use by the CPU
· Temporary (volatile) — data is lost when power is off
· Much faster to access than secondary storage
· The CPU reads from and writes to RAM constantly

The more RAM a computer has, the more programs it can run at the same time without slowing down.

Volatile VOLATILE
Contents disappear when power is removed. This is why you must save files.
Fast Access
Much faster than SSD/HDD. The CPU can read it billions of times per second.

// Slide 8 · Long-Term Storage

Secondary Storage
— Permanent Memory

What it is NON-VOLATILE
· Stores data permanently — survives power off
· Holds the operating system, programs, and user files
· Slower than RAM but far larger capacity
💿 HDD
Hard Disk Drive. Spinning magnetic platters. Cheaper, larger capacity, slower, fragile.
⚡ SSD
Solid State Drive. Flash memory chips. Faster, lighter, more durable, more expensive.

Everything you care about persisting lives here — OS, apps, documents, photos. RAM forgets; storage remembers.

// Slide 9 · Storage Locations

Internal vs. External Storage

🖥️ Internal Storage
· Inside the computer case
· Connected directly to the motherboard
· Stores the OS, installed programs, personal files
· Examples: Internal SSD, Internal HDD

Fast access. Not portable. The computer's primary drive.
🔌 External Storage
· Outside the computer
· Connected via USB, Thunderbolt, or wirelessly
· Used for backups, file transfer, extra space
· Examples: USB flash drive, external HDD/SSD

Portable. Slower connection than internal. Plug and play.
☁️ Cloud Storage — a third option
Data stored on remote servers accessed via the internet. Examples: Google Drive, iCloud, OneDrive. No physical media needed — data accessible from anywhere.

// Slide 11 · File Size

Compression — Reducing File Size

❌ Lossy Compression
· Some data is permanently removed
· Smaller file size
· Cannot restore the original exactly
· Acceptable when small quality loss is OK

Examples: JPEG, MP3, MP4
✅ Lossless Compression
· No data is lost
· Can restore original file exactly
· Larger file than lossy (but smaller than original)

Examples: PNG, ZIP, FLAC, GIF

The Boot Sequence

// Slide 13 · CPU Operation

Fetch–Decode–Execute

The CPU repeats this cycle billions of times per second. Every program instruction goes through it.

01
Fetch
Get the next instruction from memory.

PC → address of next instruction
PC → MAR → memory
Data returned to MDR
Instruction copied to IR
PC increments (+1)
02
Decode
The Control Unit interprets the instruction in IR.

What operation is this?
Add? Compare? Move? Jump?

CU sends signals to the correct parts of the CPU to prepare for execution.
03
Execute
The ALU performs the operation.

Result stored in ACC
or written back to memory via MDR.

Then: return to Fetch.
The cycle never stops.
FETCH → DECODE → EXECUTE → FETCH × billions/sec
// Slide 14 · The Big Idea

A Computer is a System

💾 Storage
Holds data permanently. The OS lives here before boot. Non-volatile.
⚡ RAM
Holds active data. The CPU's working space. Fast but volatile.
🧠 CPU
Processes instructions via Fetch–Decode–Execute. Controls everything.
🔗 Buses
Connect everything. Data, Address, and Control buses carry information between components.

👉 Everything must work in sequence. One missing component and the system fails to start.
