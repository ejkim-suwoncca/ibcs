// A2.2.1

What is a Topology?

A network topology describes the physical or logical arrangement of devices and how they are connected.

Physical Topology
The actual layout of cables and hardware in the real world.
Logical Topology
How data actually flows through the network, which may differ from physical layout.
Why It Matters
Topology affects reliability, cost, performance, and how easy it is to expand or fix the network.

IB SL requires: star, mesh, and hybrid. You must be able to draw these, describe how they work, and evaluate them against: reliability, transmission speed, scalability, data collisions, and cost.

// A2.2.1 · Star Topology

Star Topology

How it works
All devices connect to a central hub or switch. All data passes through the central device.
Evaluation
✓ Easy to add/remove devices — plug in to switch
✓ Fault isolation — one cable fails, only that device is affected
✓ No collisions — switch sends to specific device
✓ Easy to monitor / manage
✗ Single point of failure — if the switch fails, the whole network goes down
✗ Cable cost — every device needs a cable to the centre

Most common in practice — used in homes, schools, and most offices because of its simplicity and fault isolation.

// A2.2.1 · Mesh Topology

Mesh Topology

How it works
Every device connects to multiple (or all) other devices. Data can travel via multiple routes. A full mesh connects every device to every other.
Evaluation
✓ Highly reliable — multiple paths, no single point of failure
✓ Redundancy — if one link fails, traffic reroutes
✓ High throughput — parallel paths possible
✗ Expensive — many cables and connections needed
✗ Complex — difficult to set up and manage
✗ Scalability — adding a device requires many new connections

Used in: military networks, internet backbone infrastructure, and situations where reliability is more important than cost. For n nodes, full mesh needs n(n-1)/2 connections.

// A2.2.1 · Hybrid Topology

Hybrid Topology

A combination of two or more topologies — the most common in real-world networks.

How it works
Different parts of the network use different topologies, connected together.

Example: A school with star-topology in each classroom, connected via a mesh backbone between buildings.
Evaluation
✓ Flexible — each segment uses the best topology for its needs
✓ Scalable — easy to add new segments
✓ Fault tolerant — combines strengths of each topology
✗ Complex — harder to design and manage
✗ Cost — can be expensive to implement

Real world: The internet itself is a hybrid topology — local ISP networks are often star or bus; the backbone between data centres uses full mesh for maximum redundancy.

// A2.2.3 · Networking Models

Client–Server Model

A central server provides resources or services; clients request them.

How it works
The server is always on, stores data, and processes requests.
Clients connect to the server to access resources.

Client sends a request → Server processes → Server sends response.

Examples: Web browsing (HTTP), email (IMAP), online banking
Advantages & Disadvantages
✓ Centralised data — easy backup and management
✓ Security — access control on the server
✓ Scalable — server can be upgraded independently
✓ Consistent data — one source of truth
✗ Single point of failure — server goes down = no service
✗ Cost — expensive to maintain powerful server hardware
✗ Bottleneck — heavy load can slow response
// A2.2.3 · Networking Models

Peer-to-Peer Model

Every device is both a client and a server — peers share resources directly with each other.

How it works
No central server. Each peer shares its own resources (files, processing power) and uses others' resources.

Examples: BitTorrent, blockchain networks, early Skype, home file sharing
Advantages & Disadvantages
✓ No single point of failure — network continues even if nodes leave
✓ Low cost — no dedicated server needed
✓ Scalable — more peers = more capacity
✗ Security — hard to control what is shared
✗ Data inconsistency — no central version of truth
✗ Performance varies — depends on peers being online

Key comparison question: "Justify which model is more suitable for a school network." Answer: Client-server — centralised control, security, and consistent data access for students and teachers.

// A2.2.4 · Network Segmentation

Network Segmentation

Dividing a network into smaller parts to improve performance and security.

Why Segment?
Reduce congestion — fewer devices per segment = less competition for bandwidth

Improve security — limit which devices can communicate

Manage resources — different segments can have different policies
Subnetting
Dividing an IP address space into smaller subnets.

Each subnet acts as its own mini-network.

E.g., 192.168.1.0/24 split into 192.168.1.0/25 and 192.168.1.128/25

Reduces broadcast traffic within each subnet.
VLANs
Virtual Local Area Networks — logically separate networks on the same physical hardware.

Devices on the same VLAN communicate as if on the same network, even if physically apart.

Traffic between VLANs requires a router.

School example: Staff, students, and guests all connect to the same physical network, but VLANs keep their traffic separate. Staff can access the admin server; students and guests cannot.

Segmentation Benefits
Limits the spread of malware within the network
Reduces the impact of broadcast storms
Allows different security policies per segment
Improves overall network performance
VLAN vs Physical Separation
Physical separation requires separate hardware (switches, cabling)
VLANs achieve logical separation on shared hardware
VLANs are more cost-effective and flexible
Configuration-based, not hardware-based
// A2.2.2 · HL — Types of Servers

Types of Servers

Servers provide specific services to clients on a network.

DNS Server
Domain Name Server — translates human-readable domain names (e.g., google.com) into IP addresses (e.g., 142.250.80.46). Like a phone book for the internet.
DHCP Server
Dynamic Host Configuration Protocol — automatically assigns IP addresses, subnet masks, and default gateway to devices joining the network.
File Server
Stores and provides access to files for network users. Centralises storage so users can access their files from any device on the network.
Mail Server
Sends and receives email. Uses SMTP (sending) and IMAP/POP3 (receiving) protocols. Stores emails for users.
Proxy Server
Acts as an intermediary between clients and the internet. Can cache content (faster re-access), filter requests (content policy), and hide client IP addresses.
Web Server
Hosts websites and serves web pages to clients via HTTP/HTTPS. Receives requests and sends back HTML, CSS, JavaScript, and other resources.

HL Evaluation factors: When choosing a server type, consider function (what service is needed?), scalability (how many users?), reliability (what is the uptime requirement?), and security (what data does it hold?).

// DNS Deep Dive

How DNS Works

You type google.com — how does your computer find it?

1
Browser cache check — Has your browser already resolved this domain recently? Use cached IP address.
2
OS cache check — Has the operating system resolved it before? Check local DNS cache.
3
Query local DNS resolver — Your router/ISP's DNS server checks its cache. If found, return the IP.
4
Recursive resolution — If not cached, the resolver queries root name servers → TLD servers (.com) → authoritative name servers for google.com.
5
IP address returned — 142.250.x.x is sent back to your browser, which then connects to the web server at that address.

This entire process typically takes milliseconds. Cached results skip most steps. DNS resolution happens before any HTTP request.

// Watch Out

Common Errors

✗
Saying a mesh topology has no single point of failure is true — but not saying why: multiple paths mean traffic can reroute around any failed node.
✗
Confusing VLAN and subnetting — subnetting divides IP address space; VLANs divide the network logically at the data link layer. They are complementary, not the same.
✗
Saying P2P is always better because it has no SPOF — not justifying the tradeoff. P2P sacrifices security and data consistency for resilience and low cost.
✗
Confusing DNS and DHCP — DNS translates domain names to IPs; DHCP assigns IP addresses to devices. Both are essential but do very different jobs.
✗
Describing star topology as having no SPOF — the central switch IS a single point of failure. If it fails, all connected devices lose connectivity.
// A2.2 · Summary

Key Takeaways

Topologies
Star — central switch/hub; easy to manage; switch = SPOF
Mesh — every device connected; very reliable; expensive
Hybrid — mix of topologies; flexible; complex
Network Models
Client-server — centralised, secure, reliable but costly and has SPOF
P2P — decentralised, resilient, cheap but insecure and inconsistent
Segmentation
Segmenting — divides network for performance and security
Subnetting — divides IP address space
VLAN — logical separation on shared hardware
Key Servers (HL)
DNS — domain name → IP address
DHCP — auto-assigns IPs
Proxy — intermediary, caching, filtering
Web — serves HTTP content
