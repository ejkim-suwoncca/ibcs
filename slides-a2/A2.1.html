// Foundation

What is a Network?

A network is a collection of devices connected together to share data and resources.

Why Networks?
Share resources — printers, files, storage

Communicate — email, messaging, video calls

Access the internet — web, cloud services

Centralise data — one server, many users
Two Ends of Every Connection
Client — device that requests data

Server — device that provides data or services

Most devices play both roles at different times.

E.g. Your laptop is a client to YouTube's servers.

Networks range from two computers sharing a cable to billions of devices connected via the internet. Same principles, very different scale.

// A2.1.1 · Network Types

Types of Networks

Type Full Name Range / Scale Example
LAN Local Area Network Single building or campus School network, home Wi-Fi
WAN Wide Area Network Cities, countries, worldwide The internet, bank network
PAN Personal Area Network ~10 metres from a person Bluetooth headphones, smartwatch
VPN Virtual Private Network Any distance (encrypted tunnel) Remote work, secure browsing

VPN key concept: A VPN doesn't change the physical network — it creates an encrypted tunnel over an existing network, making public traffic appear private and secure.

LAN Characteristics
High speed (up to 10 Gbps+)
Owned and managed by one organisation
Low latency
Usually wired (Ethernet) or Wi-Fi
WAN Characteristics
Spans large geographic distances
Often owned by ISPs or telecoms
Higher latency than LAN
Uses leased lines, fibre, satellite
// A2.1.2 · Digital Infrastructure

Modern Digital Infrastructure

The physical and virtual systems that enable our connected world.

🌐 The Internet
A global WAN connecting billions of devices.

Uses TCP/IP protocols.
The WWW (websites) runs on top of the internet — they are not the same thing.

Benefit: Universal connectivity
Limitation: No central control → security risks
☁️ Cloud Computing
Renting computing resources over the internet instead of owning physical hardware.

Benefit: Scalable, cost-effective, accessible anywhere
Limitation: Internet-dependent, data privacy concerns

Examples: Google Drive, AWS, Netflix
🔗 Distributed Systems
Multiple computers work together, appearing as one system.

Benefit: Fault tolerant, scalable
Limitation: Complex to build and maintain

Examples: Cryptocurrency blockchain, Google Search
📡 Edge Computing
Processing data near where it is collected, not in a central cloud.

Benefit: Low latency, works offline
Limitation: More hardware needed at edge

Examples: Smart traffic lights, autonomous vehicles

Mobile Networks (4G/5G) — wireless infrastructure enabling devices to connect from anywhere. 5G offers dramatically lower latency and higher bandwidth than 4G.

// A2.1.3 · Network Devices

Network Devices

Router
Directs data packets between different networks (e.g., your home LAN to the internet). Reads IP addresses to choose the best path.
Switch
Connects devices within a LAN. Reads MAC addresses to deliver frames only to the intended device — more efficient than a hub.
Modem
Modulates/demodulates signals. Converts digital data to signals suitable for the transmission medium (e.g., telephone lines, cable) and back.
Gateway
Connects networks using different protocols. Translates between them. Your home router often acts as a gateway to the internet.
Firewall
Inspects and filters incoming and outgoing traffic based on rules. Blocks unauthorised access. Can be hardware or software.
NIC
Network Interface Card — the hardware component that enables a device to connect to a network. Has a unique MAC address burned in at manufacture.
WAP
Wireless Access Point — allows Wi-Fi devices to connect to a wired network. Extends network coverage wirelessly.
// Key Distinction

Router vs Switch

A common exam question — know the difference.

🔀 Router
Works at the network layer
Reads IP addresses
Connects different networks
Chooses the best route/path for data

Analogy: Post office sorting data by city/country
🔄 Switch
Works at the data link layer
Reads MAC addresses
Connects devices within one network
Sends data only to the intended device

Analogy: Office receptionist routing calls to correct desk

Common Error: Students confuse switches and hubs. A hub broadcasts to all devices; a switch sends to the specific destination. Switches are far more efficient.

// A2.1.4 · Protocols

What are Protocols?

A protocol is a set of agreed rules that defines how data is formatted, transmitted, and received between devices.

Without Protocols
Different devices from different manufacturers would not be able to understand each other. Data would be meaningless noise.
With Protocols
Any device that implements the same protocol can communicate — regardless of hardware, OS, or manufacturer.
Real World Analogy
Like a shared language. If both speakers use English, they can communicate. If one uses Korean and the other uses Arabic, they cannot.

Protocols define: packet structure, error checking, handshaking procedures, data flow control, and connection management.

// A2.1.4 · Key Protocols

Essential Protocols

Protocol Full Name Layer Purpose
TCP Transmission Control Protocol Transport Reliable, ordered delivery. Checks all data arrived correctly. Slower.
UDP User Datagram Protocol Transport Fast, connectionless. No guarantee of delivery. Used for streaming, gaming.
HTTP Hypertext Transfer Protocol Application Transfers web pages. Data is unencrypted.
HTTPS HTTP Secure Application HTTP + TLS encryption. Secure web browsing. Look for the 🔒 in browsers.
DHCP Dynamic Host Configuration Protocol Application Automatically assigns IP addresses to devices on a network.
TCP Use Cases
Emails, web browsing, file downloads
— where data integrity is critical.
All data must arrive correctly.
UDP Use Cases
Video streaming, online gaming, VoIP
— where speed matters more than perfection.
A few lost packets = minor glitch, not a crash.
// Deep Dive

TCP vs UDP

TCP — Reliable
Connection-oriented — must establish connection first (3-way handshake: SYN → SYN-ACK → ACK)

Guarantees delivery — lost packets are retransmitted

Ordered — packets arrive in the correct sequence

Slower due to overhead of checking and acknowledging

Used for: HTTP/S, email (SMTP), file transfer (FTP)
UDP — Fast
Connectionless — just fires packets, no setup

No guarantee — packets may be lost, duplicated, or arrive out of order

No retransmission of lost packets

Faster — minimal overhead

Used for: Live video, DNS lookups, online gaming, VoIP

Exam tip: Always justify your choice. If a question asks which protocol to use, state the reason — e.g. "TCP because file integrity is essential and losing data would corrupt the file."

// A2.1.4 · DHCP

How DHCP Works

Every device on a network needs a unique IP address. DHCP assigns them automatically.

1
DISCOVER — New device joins network and broadcasts: "Is there a DHCP server? I need an IP address."
2
OFFER — DHCP server responds with an available IP address and lease time: "Here, take 192.168.1.42 for 24 hours."
3
REQUEST — Device officially requests that specific IP address.
4
ACKNOWLEDGE — Server confirms: "192.168.1.42 is yours until it expires."

Without DHCP: Every device would need a manually configured IP address — impractical for networks with hundreds or thousands of devices (e.g., a school or hotel).

// Common Misconception

Internet ≠ World Wide Web

The Internet
The physical infrastructure — cables, routers, servers, and the protocols (TCP/IP) that connect them.

Like the road network — the roads themselves.

Also carries email, VoIP, FTP, streaming, gaming…
The World Wide Web (WWW)
A service that runs over the internet — the collection of websites and web pages accessed via HTTP/S.

Like the cars and lorries using the road network.

Invented by Tim Berners-Lee in 1989.

Key point: The Web is just one of many services that use the internet. If you send an email, you use the internet but not the Web (unless you use webmail).

Other internet services: Email (SMTP/IMAP), file transfer (FTP), streaming (RTP/UDP), gaming, VoIP (Skype, WhatsApp calls).

// Watch Out

Common Errors

✗
Confusing the Internet with the WWW — the internet is the infrastructure; the Web is a service running on it.
✗
Saying a switch connects different networks — switches connect devices within a network; routers connect different networks.
✗
Saying UDP is better than TCP — neither is universally better; the right protocol depends on the application and whether reliability or speed is the priority.
✗
Describing a VPN as a different network — a VPN creates an encrypted tunnel over an existing network; it does not create new physical infrastructure.
✗
Confusing DHCP and DNS — DHCP assigns IP addresses to devices on a network; DNS translates domain names to IP addresses (covered in A2.2).
