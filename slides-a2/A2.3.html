// A2.3.1 · IP Addressing

What is an IP Address?

An IP address is a unique numerical label assigned to each device on a network — like a postal address for data.

IPv4
32-bit address
Written as 4 octets: 192.168.1.42
Each number: 0–255
~4.3 billion unique addresses

Problem: Not enough for every device on Earth!
IPv6
128-bit address
Written in hexadecimal:
2001:0db8:85a3::8a2e:0370:7334
~340 undecillion unique addresses

Solution: Essentially unlimited addresses for all future devices

Key IPv4 vs IPv6 differences: IPv4 = 32-bit, decimal notation, ~4.3B addresses. IPv6 = 128-bit, hexadecimal notation, ~3.4×10³⁸ addresses. IPv6 also has built-in security (IPsec) and simpler routing headers.

// A2.3.1 · IP Address Types

IP Address Categories

Public IP Address
Assigned by your ISP. Visible on the internet.

Globally unique — no two devices on the internet share the same public IP.

Example: Your home router's public IP seen by websites you visit.
Private IP Address
Used within a private network (home, office). NOT routable on the public internet.

Reserved ranges: 192.168.x.x, 10.x.x.x, 172.16.x.x

Example: Your laptop's IP address at home: 192.168.1.5
Static IP Address
Manually assigned — never changes.

Benefits: Reliable for servers, consistent address for remote access
Limitation: Manual configuration, costs more from ISP
Dynamic IP Address
Assigned by DHCP — may change each time you connect.

Benefits: Efficient use of IP pool, automatic, no admin needed
Limitation: Address may change — bad for hosting servers
NAT — Network Address Translation
NAT allows many private IP devices to share one public IP address. The router keeps a translation table mapping internal:port ↔ external:port.

Benefits: Conserves IPv4 addresses, hides internal network structure (security through obscurity).
How: When a device sends data out, the router replaces the private source IP with its public IP. When data returns, the router translates it back to the correct private IP.
// A2.3.2 · Transmission Media

Transmission Media

The physical medium through which data is transmitted. Each has tradeoffs.

🔆 Fibre Optic
Transmits light pulses through glass or plastic fibre.

✓ Very high bandwidth (Tbps possible)
✓ Very low signal loss over long distances
✓ No electromagnetic interference
✓ Secure (hard to tap)
✗ Expensive to install
✗ Fragile — glass can break
✗ Complex connections
🔌 Twisted Pair (Copper)
Two insulated copper wires twisted together. The twisting reduces crosstalk.

✓ Inexpensive
✓ Easy to install
✓ Compatible with existing infrastructure
✗ Limited bandwidth compared to fibre
✗ Signal degrades over ~100m
✗ Susceptible to EMI
✗ Easier to tap
📡 Wireless
Radio waves (Wi-Fi, Bluetooth, cellular).

✓ No cables needed — mobility
✓ Easy to expand
✓ Low installation cost
✗ Interference from walls, devices
✗ Lower reliability than wired
✗ Security risk — signals can be intercepted
✗ Limited range

CostHighLowLow–Medium

// A2.3.3 · Packet Switching

Why Packet Switching?

The alternative is circuit switching — but the internet uses packet switching. Why?

Circuit Switching (Old)
A dedicated path is established before data is sent and held open for the entire connection.

Like making a phone call — the line is reserved for you.

✗ Inefficient — line is occupied even during silence
✗ Not scalable for millions of simultaneous connections
Packet Switching (Internet)
Data is split into packets, each sent independently through the network.

Different packets may take different routes to the same destination.

✓ Efficient — bandwidth is shared dynamically
✓ Resilient — packets reroute around failures
✓ Scalable — millions of users can share infrastructure

Packet switching was developed by Paul Baran (RAND Corporation) and Donald Davies (NPL) in the 1960s, originally for military networks that needed to survive nuclear attacks.

// A2.3.3 · Packet Switching

How Packet Switching Works

1
Segmentation — The sender's OS splits the data into small chunks called packets (typically 1,500 bytes for Ethernet). Each packet contains a portion of the data.
2
Header added — Each packet gets a header containing: source IP, destination IP, packet number (sequence), total number of packets, and error-checking data (checksum).
3
Independent routing — Packets are sent into the network. Each router reads the destination IP and forwards the packet to the next best hop. Different packets may take different routes.
4
Arrival — Packets may arrive out of order or some may be delayed. Routers don't guarantee delivery order.
5
Reassembly — The receiving device uses the sequence numbers in the headers to reassemble packets in the correct order. If TCP is used, missing packets are requested again.
Packet Structure
[HEADER] Source IP | Dest IP | Seq# | Checksum
[PAYLOAD] Actual data (up to ~1,460 bytes)
[TRAILER] Error detection (CRC)
Role of Devices
Switch — forwards packets within a LAN using MAC addresses
Router — forwards packets between networks using IP addresses; chooses the best path
// Visual Explanation

Packets Take Different Paths

Key insight: Packets 1 and 2 are part of the same message but take completely different routes. The receiver reassembles them in the correct order using sequence numbers. This is why the internet is robust — if one path fails, traffic uses another.

// Key Concepts

Bandwidth & Attenuation

Bandwidth
The maximum amount of data that can be transmitted over a medium per unit of time.

Measured in: bits per second (bps), Mbps, Gbps, Tbps

Higher bandwidth = more data simultaneously = faster downloads

Analogy: The width of a pipe — wider = more water flows
Attenuation
The weakening of a signal as it travels through a medium over distance.

All media suffer from attenuation — some more than others.

Twisted pair: significant attenuation (~100m max for Ethernet)
Fibre optic: very low attenuation (many km without repeaters)
Wireless: high attenuation through walls/obstacles
Latency
The delay between sending and receiving data. Measured in milliseconds (ms).

Affected by: distance, number of hops, router processing time, congestion.

Important for: gaming, video calls, real-time applications.
Throughput
The actual data transfer rate achieved — always lower than theoretical bandwidth due to protocol overhead, congestion, errors, and retransmissions.
// A2.3.4 · HL — Routing

Static vs Dynamic Routing

How do routers decide which path to send a packet on?

Static Routing
Routes are manually configured by a network administrator and don't change automatically.

✓ Predictable — routes are always the same
✓ No overhead — no routing protocol traffic
✓ Secure — no automatic route changes
✓ Simple for small networks
✗ Manual updates required when topology changes
✗ Does not adapt to failures automatically
✗ Scales poorly on large networks
Dynamic Routing
Routers automatically discover routes and update their routing tables using routing protocols (e.g., OSPF, BGP — protocol details not required).

✓ Adapts to failures — automatically finds new routes
✓ Scales well on large networks
✓ Low admin overhead once set up
✗ Protocol overhead — routers exchange routing info, consuming bandwidth
✗ Complex to configure and troubleshoot
✗ Convergence time — takes time after a failure to settle
FactorStaticDynamic
ConfigurationManualAutomatic
MaintenanceHigh (manual updates)Low (self-updating)
Network sizeSmallLarge
Failure responseManual interventionAutomatic reroute
Resource usageLowHigher (protocol traffic)
// Watch Out

Common Errors

✗
Saying all packets take the same path — in packet switching, each packet is independently routed and may take a different path.
✗
Confusing bandwidth and speed — bandwidth is the theoretical maximum capacity; actual transfer speed depends on many factors and is usually lower.
✗
Not mentioning NAT's security benefit — NAT hides internal IP addresses from the external network, providing a layer of security as well as address conservation.
✗
Saying wireless is always worse — in some scenarios (mobility, ease of installation) wireless is the better choice. Always justify based on context.
✗
Saying IPv6 is the same as IPv4 but longer — IPv6 also introduced auto-configuration, improved routing headers, built-in IPsec, and elimination of NAT as a workaround.
// Real World

What Happens When You Visit a Website?

1
Your device has a private IP (e.g., 192.168.1.5) assigned by DHCP. Your router has a public IP.
2
You type example.com. Your device sends a DNS query to resolve the domain to an IP address (e.g., 93.184.216.34).
3
Your browser creates an HTTP/S request using TCP. The data is split into packets with headers containing source/destination IPs.
4
The router performs NAT, replacing your private source IP with the public IP, then forwards packets. Routers along the way forward packets hop by hop.
5
The web server receives packets via fibre optic or copper cables, assembles them, and sends the web page content back — in packets — via the same process in reverse.

Key Takeaways

IP Addressing
IPv4 — 32-bit, 4.3B addresses, running out
IPv6 — 128-bit, essentially unlimited
Public — internet-visible, unique globally
Private — internal network only (192.168.x.x)
NAT — maps private IPs to one public IP
Transmission Media
Fibre — fastest, longest, most expensive
Twisted pair — cheap, common, ~100m limit
Wireless — mobile, interference-prone, insecure
Packet Switching
Data split into packets with headers
Packets route independently — may take different paths
Receiver reassembles using sequence numbers
Resilient: packets reroute around failures
Routing (HL)
Static — manual, predictable, small networks
Dynamic — automatic, adaptive, large networks
Tradeoff: control and simplicity vs resilience and scale
