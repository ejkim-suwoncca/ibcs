What IB Requires HL Only

2.1.5
Describe the function of the TCP/IP model HL
Application, transport, internet, network interface — the role of each layer and how they interact to ensure reliable data transmission

Context: The TCP/IP model is a conceptual framework for understanding how networking communication is organized. It largely supersedes the older OSI model. Devices and protocols can be mapped to specific layers, which makes troubleshooting and design systematic.

Common mistake: It's easy to confuse which protocols operate at which layer. Keep a clear map: HTTP, SMTP, DNS → Application; TCP, UDP → Transport; IP → Internet; Ethernet, WiFi, MAC → Network Interface.

// Motivation · Why a Layered Model?

Why Layers?

Networking is complex. A layered model breaks the problem into smaller, independent responsibilities — each layer does one job and talks only to the layers directly above and below it.

Modularity
Each layer can be developed, updated or replaced independently without affecting other layers. E.g., WiFi can replace Ethernet at the network interface layer while everything above stays the same.
Abstraction
Higher layers don't need to know how lower layers work. A web browser (application layer) just sends an HTTP request — it doesn't care if the physical medium is fibre, copper or wireless.
Interoperability
Any device that follows the TCP/IP standard can communicate with any other, regardless of manufacturer or operating system. This is why a Mac can browse a website on a Windows server.

Analogy: Think of a postal service. The post office doesn't care what's inside the envelope (abstraction). The envelope, sorting system and delivery truck are separate layers — each has its own job, but they work together to get the letter delivered.

// Layer 4 · Application Layer

Application Layer

The topmost layer — where actual applications and end-user protocols operate. Responsible for correctly forming and interpreting messages.

HTTP / HTTPS
HyperText Transfer Protocol — used for web browsing. HTTPS adds TLS encryption. Port 80 (HTTP) and 443 (HTTPS). A browser sends an HTTP GET request; the server responds with a web page.
SMTP / POP3 / IMAP
Email protocols. SMTP (port 587) sends outgoing email. POP3 downloads email to a device. IMAP synchronizes email across devices from the server.
FTP / SFTP
File Transfer Protocol — transfers files between client and server. SFTP adds SSH encryption for secure transfers. Used in web hosting, data migration.
DNS
Domain Name System — translates human-readable domain names (e.g. bbc.co.uk) into IP addresses. Without DNS, users would need to remember IP addresses to visit websites.
SSH (Secure Shell)
Encrypted remote access protocol. Allows secure login to remote servers, command execution, and tunnelling. Port 22. Replaces older insecure protocols like Telnet.
DHCP
Dynamic Host Configuration Protocol — automatically assigns IP addresses to devices joining a network, so they don't need to be configured manually.

Key idea: The application layer protocols define the format and rules for specific types of communication. The layer does not deal with how data physically gets there — that's the job of the lower layers.

// Layer 3 · Transport Layer

Transport Layer

Responsible for getting data from the correct source application to the correct destination application. Uses port numbers as addresses within a device.

TCP — Transmission Control Protocol
Connection-oriented. Establishes a connection before sending (3-way handshake: SYN → SYN-ACK → ACK).

Guarantees: delivery, correct order, error checking (checksum). Lost packets are retransmitted.

Use when: reliability matters more than speed — web browsing, email, file transfer, banking.
UDP — User Datagram Protocol
Connectionless. Data is fired off without establishing a connection first. No guarantee of delivery, order, or error correction.

Faster and lower overhead than TCP. Packet loss is acceptable.

Use when: speed matters more than reliability — video streaming, online gaming, live audio, DNS lookups.
Port Numbers
Port numbers identify which application on a device should receive the data. Think of them as apartment numbers within a building address (IP). Common defaults: HTTP 80, HTTPS 443, SMTP 587, SSH 22, DNS 53.
Segmentation
The transport layer breaks data from the application layer into smaller segments (TCP) or datagrams (UDP) that can fit into IP packets. These are reassembled at the destination in the correct order (TCP only).
// Layer 2 · Internet Layer

Internet Layer

Responsible for addressing, packaging, and routing data across multiple networks to reach the correct destination device.

IP Packets
Receives segments from the transport layer and wraps them in IP packets. Each packet has a source IP address and destination IP address in its header. Packets may travel different routes and arrive out of order.
Routing
Routers operate at this layer. They examine the destination IP address in each packet and forward it along the best path through the network towards its destination, potentially crossing many networks.

IPv4 vs IPv6: IPv4 uses 32-bit addresses (e.g. 192.168.1.1). IPv6 uses 128-bit addresses (e.g. 2001:0db8:85a3::8a2e:0370:7334) to solve address exhaustion. Both operate at the internet layer.

ICMP: The Internet Control Message Protocol also lives at this layer. It is used for diagnostic messages — e.g. the ping command uses ICMP echo requests to test whether a host is reachable.

// Layer 1 · Network Interface Layer

Network Interface Layer

The lowest layer — handles the physical transmission of bits over the actual medium (cable, radio waves). Uses MAC addresses for local delivery.

Physical Transmission
Takes IP packets from the internet layer and converts them into frames for transmission over the physical medium (electrical signals on copper, light pulses on fibre, radio waves for WiFi). Receives frames and converts them back into packets for the internet layer.
MAC Addresses
Hardware addressing uses MAC (Media Access Control) addresses — unique 48-bit identifiers burned into every NIC at manufacture. MAC addresses are used for local delivery within a network segment. Switches use MAC tables to direct frames.
Ethernet
The most common wired LAN standard. Operates at this layer, using frames to package data for transmission over twisted-pair or fibre cables.
WiFi (802.11)
Wireless transmission standard. Uses radio frequencies (2.4 GHz, 5 GHz) instead of cables. Wireless access points connect devices to the network.
Devices
Network interface cards (NICs), modems, switches, and wireless access points all operate primarily at this layer.
// Encapsulation · How Data Travels Down

Encapsulation

As data moves down the stack on the sending device, each layer adds its own header (and sometimes a trailer). This wrapping process is called encapsulation.

App
Application layer creates the message. e.g. an HTTP request: GET /index.html HTTP/1.1. This is the payload — the actual data being sent.
Trans
Transport layer adds a segment header. Contains source and destination port numbers, sequence number (TCP), checksum. The message becomes a TCP segment or UDP datagram.
Net
Internet layer adds an IP header. Contains source and destination IP addresses, TTL (time to live), protocol type. The segment becomes an IP packet.
NI
Network interface layer adds a frame header and trailer. Contains source and destination MAC addresses, frame type, and a checksum (CRC). The packet becomes a frame, ready for physical transmission.

Decapsulation: At the receiving end, each layer strips off its header as data travels up the stack, until the original message arrives at the application. Each layer reads only the headers it added.

// Device-to-Layer Mapping

Which Device Operates Where?

IB often asks you to map network hardware to the correct layer. Knowing this aids troubleshooting.

Application Layer
Gateway — connects two networks using different protocols; translates between them. Operates at the application layer because it must understand the application-level data to perform translation.
Transport + Internet Layers
Hardware firewall — inspects and filters traffic based on IP addresses and port numbers (transport layer info). Operates across both transport and internet layers.
Internet Layer
Router — uses IP addresses to forward packets between networks. Makes routing decisions based on internet-layer headers. Operates at the internet layer.
Network Interface Layer
Switch, NIC, modem, wireless access point — all operate at the network interface layer, using MAC addresses and managing physical transmission within a local network segment.

Exam tip: Routers = internet layer (IP addresses). Switches = network interface layer (MAC addresses). Gateways = application layer (protocol translation). Firewalls = transport + internet layer (IPs + ports).

// Review · A2.1.5

Summary & Key Takeaways

Application Layer (4)
User-facing protocols: HTTP/HTTPS (web), SMTP/POP3 (email), FTP/SFTP (files), DNS (name lookup), SSH (remote access), DHCP (IP allocation). Formats the message.
Transport Layer (3)
TCP — connection-oriented, reliable, ordered delivery. UDP — connectionless, fast, no guarantee. Uses port numbers to route data to the correct application on a device.
Internet Layer (2)
IP — encapsulates data into packets with source/destination IP addresses. Handles routing across multiple networks. Routers operate here.
Network Interface Layer (1)
Physical transmission — converts packets to frames and bits. Uses MAC addresses for local delivery. Ethernet, WiFi, switches, NICs all live here.
Encapsulation
Each layer adds a header as data goes down the stack. Each layer strips its header as data goes up the stack at the destination. Only the correct layer reads its own header.
Device Mapping
Router → Internet; Switch/NIC/WAP/Modem → Network Interface; Firewall → Transport + Internet; Gateway → Application. Know these for the exam.

Common mistake to avoid: Don't confuse port numbers (transport layer, identifies application) with IP addresses (internet layer, identifies device) with MAC addresses (network interface layer, identifies NIC). Each solves a different addressing problem at a different layer.
