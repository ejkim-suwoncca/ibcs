// B2.1.1 · Variables & Data Types

Five Data Types You Must Know

A variable is a named memory location that stores one value. The data type tells you what kind of value it holds and what operations are valid.

Boolean
True or False only.

passed = True
integer
Whole numbers.

score = 87
decimal / float
Numbers with a fractional part.

avg = 91.5
string
Sequence of characters.

name = "Mina"
char — single character
One letter, digit or symbol. Python does not separate char from string: "A" is a one-character string. Java uses char grade = 'A';

Guide requirement: The five data types above are explicitly listed. A wrong data type causes type errors — for example, trying to add an integer to a string.

// B2.1.1 · Boolean Variables

Boolean: Store a Yes/No Answer

Use a Boolean when a program needs to remember whether something is true or false.

Coding situation
attendance_taken = True homework_done = False if homework_done: print("Submit work") else: print("Reminder needed")
Common errors
Writing "True" as a string instead of True as a Boolean.

Testing homework_done == "yes" when the variable already stores True or False.

Practice: Create a Boolean variable called has_paid. Print "Entry allowed" only when it is True.

// B2.1.1 · Integer Variables

Integer: Count Whole Things

Use an integer for counts, scores, ages, positions, and any value without a fractional part.

Coding situation
students = 24 absent = 3 present = students - absent print("Present:", present)
Common errors
Reading input as text and forgetting to convert it:

age = input("Age: ") stores a string.
age = int(input("Age: ")) stores an integer.

Practice: Store the number of questions and the number answered correctly. Calculate the number answered incorrectly.

// B2.1.1 · Decimal / Float Variables

Float: Store Measurements and Averages

Python calls decimal numbers float. Use float when a value can include digits after the decimal point.

Coding situation
quiz1 = 88.5 quiz2 = 91.0 average = (quiz1 + quiz2) / 2 print("Average:", average)
Common errors
Using int() when decimals matter. int(88.5) becomes 88 and loses the fractional part.

Use float(input("Price: ")) for decimal input.

Practice: Store the price of one item and the tax rate. Calculate the final price.

// B2.1.1 · String and Char Variables

String: Store Text

Use a string for names, emails, messages, codes, and any data that should be treated as text.

Coding situation
first = "EJ" last = "Kim" email = "ej.kim@suwoncca.org" full_name = first + " " + last print(full_name) print(email)
Char in other languages
initial = "E" is still a string in Python.

Java can store one character with char initial = 'E';

Common error: Adding a string and a number directly causes an error: "Score: " + 95. Convert the number first: "Score: " + str(95).

// B2.1.2 · Substrings

Extracting and Manipulating Text

A substring is any part of a string. You can identify, extract, alter, concatenate or replace substrings using indexing and built-in methods.

Indexing & Slicing (Python)
email = "ej.kim@suwoncca.org" # 0123456789... user = email[:6] # "ej.kim" domain = email[7:] # "suwoncca.org" tld = email[-3:] # "org" one_ch = email[2] # "."
Concatenation & Methods (Python)
first = "EJ" last = "Kim" full = first + " " + last # concatenate email2 = email.replace("suwoncca", "ib") tag = user.upper() # "EJ.KIM" clean = " hello ".strip() # "hello"
OperationPython syntaxWhat it does

// B2.3.1 · Sequence

Order of Instructions Changes Everything

A program executes line by line. If steps are in the wrong order, the output can be wrong even when each line looks reasonable.

Correct sequence
total = 0 # 1. initialise total = total + 5 # 2. update print(total) # 3. output: 5
Broken sequence
total = total + 5 # NameError: total not defined total = 0 print(total)

Guide requirement: B2.3.1 expects you to construct programs that use sequence correctly. Trace the values after each line to check the order.

// B2.1.3 · try · except/catch · finally

Handling Exceptions in Python and Java must include

Python — try / except / finally
try: number = int(input("Enter a number: ")) result = 10 / number print(result) except ValueError: print("Please enter a whole number.") except ZeroDivisionError: print("Cannot divide by zero.") finally: print("Attempt finished.") # always runs
Java — try / catch / finally
try { int number = scanner.nextInt(); int result = 10 / number; System.out.println(result); } catch (InputMismatchException e) { System.out.println("Enter a whole number."); } catch (ArithmeticException e) { System.out.println("Cannot divide by zero."); } finally { System.out.println("Attempt finished."); }
BlockWhat goes insideWhen it runs
tryCode that might cause an exceptionAlways attempted first
except / catchResponse to a specific exception typeOnly if that exception is raised in try
finallyCleanup code (e.g. close a file)Always — whether or not an exception occurred

Common error: Catching every error with a vague message makes debugging harder. Catch the specific exception when you know what can go wrong.

// B2.1.4 · Debugging Techniques

Finding and Fixing Errors Systematically

Debugging is the process of finding and fixing errors in code. The IB expects you to know four techniques and when to use each.

Trace table
Track every variable step by step through an algorithm on paper. Used at design stage — before code is written — to test logic.

Best for: finding logic errors in a short algorithm.
Print statements
Insert print(variable) at key points to expose the program's internal state while it runs.

Best for: quickly checking which value a variable holds at runtime.
Breakpoint debugging
Set a marker in an IDE. Execution pauses there so you can inspect variables and memory at that exact moment.

Best for: complex programs where print statements alone are not enough.
Step-by-step execution
Run code one line at a time inside an IDE debugger. Observe how variables change with each step.

Best for: finding exactly which line introduces a wrong value.

Rule: Do not guess. Collect one piece of evidence at a time. The goal is to know which line caused the problem, not just that the output was wrong.

// B2.3.2 · Selection

Making Decisions in Code

Selection structures execute different code blocks depending on whether a condition is True or False. Use Boolean operators (AND, OR, NOT) and relational operators (<, <=, >, >=, ==, !=) to build conditions.

Python — if / elif / else
score = 78 if score >= 80: print("Pass with distinction") elif score >= 50: print("Pass") else: print("Fail") # Boolean operators in conditions: if score >= 50 and score <= 100: print("Valid score, passed")
Java — if / else if / else
if (score >= 80) { System.out.println("Pass with distinction"); } else if (score >= 50) { System.out.println("Pass"); } else { System.out.println("Fail"); } // Boolean operators: if (score >= 50 && score <= 100) { System.out.println("Valid score, passed"); }

Guide requirement: Must include if, else and else if (Java) / elif (Python). Must also demonstrate use with Boolean and relational operators.

// B2.1.3 · Why Programs Fail

Three Sources of Failure must include

Failure means the program cannot complete the task correctly. Sometimes it crashes; sometimes it runs but gives the wrong result.

Unexpected input
The user enters a value the program was not prepared for, such as "ten" when a number is needed.
Resource unavailable
The program needs something it cannot access, such as a missing file, offline printer, or failed network connection.
Logic error
The instructions run, but the thinking is wrong. The program may finish with an incorrect answer.

Connection: Exception handling is one way to respond to failures that would otherwise stop the program suddenly.
