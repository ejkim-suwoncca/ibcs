"Sequential file" = records stored one after another in a fixed order. To reach record 50, you must read through records 1–49 first. This is different from direct access (random access) files.

// B2.5.1 · Sequential Text Files

What Is a Text File?

A sequential text file stores data as plain text, with each record on a separate line. Records are read or written in order from beginning to end.

Key Characteristics
Sequential access — must read from the top

Human-readable — plain text, not binary

One record per line — separated by newline (\n)

Persistent — survives after the program ends
Example File: students.txt
line 1 → Alice,95
line 2 → Bob,82
line 3 → Carol,91
line 4 → David,78

Compare with: RAM (volatile — lost when program ends), databases (structured, indexed), binary files (not human-readable). Text files are the simplest form of persistent storage.

// B2.5.2 · File Modes

Opening a File: Modes

The second argument to open() sets the mode — it controls what you can do with the file and what happens to existing content.

ModeNameWhat It DoesFile Must Exist?Existing Content
"r" Read Opens file for reading only Yes — error if missing Unchanged
"w" Write Opens file for writing — creates new file or overwrites No — creates if missing ⚠ Deleted (overwritten)
"a" Append Opens file for writing — adds to the end No — creates if missing Preserved (added to end)
"r+" Read+Write Both reading and writing Yes — error if missing Preserved (overwrite at position)

⚠ Common mistake: Using "w" when you meant "a". Opening in write mode immediately destroys all existing content — even before you write anything!

IB Exam tip: You will always be asked about "r", "w", and "a" specifically. Know the difference cold — especially that "w" wipes the file and "a" preserves it.

// B2.5.3 · Reading from a File

Reading: Three Methods

read()
Reads the entire file as one big string. Good for small files. Returns everything including newlines (\n).
readline()
Reads one line at a time. Returns an empty string "" when end of file is reached. Used in a loop.
readlines()
Reads all lines and returns a list. Each element is one line including the \n character.
# Method 1: read() — entire file as one string f = open("students.txt", "r") content = f.read() print(content) f.close() # Method 2: readline() — one line at a time (most common) f = open("students.txt", "r") line = f.readline() while line != "": print(line.strip()) # strip() removes trailing \n line = f.readline() f.close() # Method 3: readlines() — get all lines as a list f = open("students.txt", "r") lines = f.readlines() # ['Alice,95\n', 'Bob,82\n', ...] f.close()

IB preferred pattern: Use readline() in a while loop — it processes line by line without loading the entire file into memory. This is the most commonly tested approach.

// B2.5.3 · Writing to a File

Writing: write() and append

Use write() to output text to a file. You must manually add newline characters (\n) — they are not added automatically.

# Writing — creates new file (or wipes existing) f = open("students.txt", "w") f.write("Alice,95\n") f.write("Bob,82\n") f.write("Carol,91\n") f.close() # Appending — adds to end of existing file f = open("students.txt", "a") f.write("David,78\n") # added after Carol f.close() # Writing a list of records in a loop students = [("Eve", 88), ("Frank", 74)] f = open("new_students.txt", "w") for name, score in students: f.write(name + "," + str(score) + "\n") f.close()

⚠ Remember \n: Without \n, all records will be on the same line. The file would read Alice,95Bob,82 instead of being on separate lines. Always end each record with \n.

// B2.5.3 · Closing Files

Always Close Your File

Calling close() releases the file back to the OS. Without it, data may not be saved (it stays in the write buffer) and the file may be locked.

⚠ Without close()
Data may remain in the write buffer — not saved to disk

Other programs can't access the file

File may become corrupt if program crashes

OS resource (file descriptor) is leaked
✓ The "with" Statement
Python's preferred approach — automatically closes the file even if an error occurs (exception-safe):

with open("file.txt", "r") as f:
    data = f.read()
# file is closed here automatically

IB Exam note: The IB syllabus uses the explicit open() / close() pattern — both are acceptable but you will commonly see close() in exam answers. Know both.

def find_top(filename, threshold): f = open(filename, "r") line = f.readline() while line != "": parts = line.strip().split(",") name = parts[0] score = int(parts[1]) if score >= threshold: print(name, "passed") line = f.readline() f.close() find_top("grades.txt", 90)

Read 3
line = "Carol,91\n" → strip → split → name="Carol", score=91. 91 ≥ 90 → print "Carol passed"
EOF

// B2.5.3 · Tracing

Trace: Writing Records

This program collects names and scores from the user and saves them to a file:

# Collect data from user and write to file f = open("results.txt", "w") n = int(input("How many students? ")) for i in range(n): name = input("Name: ") score = input("Score: ") f.write(name + "," + score + "\n") f.close() print("Saved!")

TRACE — user enters n=2, then Alice/95, then Bob/82:

Init
open("results.txt", "w") → creates empty file. n = 2
i=0
name="Alice", score="95" → write("Alice,95\n") → file: Alice,95
i=1
name="Bob", score="82" → write("Bob,82\n") → file: Alice,95 / Bob,82
End
close() → data flushed to disk. print("Saved!") → program done.

Key detail: score is entered as a string via input() — no need to convert to int for writing. If we later read it back and need to compare numerically, then we cast with int().

// B2.5.3 · Parsing File Data

strip() and split()

Raw lines from a file include the newline character (\n) at the end. Two string methods are essential for cleaning and splitting data.

.strip()
Removes leading and trailing whitespace, including \n

"Alice,95\n".strip()
→ "Alice,95"

Use it on every line you read before processing
.split(",")
Splits a string at a delimiter into a list

"Alice,95".split(",")
→ ["Alice", "95"]

Access parts by index: parts[0], parts[1]
# Full pattern: read → strip → split → use line = f.readline().strip() # "Alice,95" parts = line.split(",") # ["Alice", "95"] name = parts[0] # "Alice" score = int(parts[1]) # 95 (as integer)

Comma-separated values (CSV) is the most common format for simple file-based records in IB questions. Always split on "," and cast numbers with int() or float() when needed.

// Common Mistakes

Common Mistakes

✗
Forgetting \n when writing — f.write("Alice,95") with no newline means the next record will be on the same line. Always write("...\n").
✗
Not calling strip() — comparing "Alice\n" == "Alice" returns False. Always strip() your line before comparing or splitting.
✗
Using "w" instead of "a" — every time you open in "w" mode the file is wiped. If you want to add a new student, use "a" mode.
✗
Not casting numbers — all data read from files is a string. If you need to do arithmetic (score > 90), cast with int(parts[1]) first.
✗
Forgetting to close() — data may not be saved to disk. The file object keeps a write buffer; close() flushes it.
✗
Wrong end-of-file check — readline() returns "" (empty string) at EOF. Don't check for None or for a specific number of lines.
// Exam Strategy

Exam Answer Checklist

When asked to write or trace file processing code, follow this pattern every time:

①
Open the file with the correct mode — "r" to read, "w" to write fresh, "a" to append. State which mode and why.
②
Read loop — use readline() in a while loop; check for "" to detect end of file. Don't use a counter unless the question specifies n lines.
③
Process each line — strip() first, then split(",") if there are multiple fields. Cast to int/float only when arithmetic is needed.
④
Write records — always append \n to end of each record. Write strings only (convert numbers with str()).
⑤
Close the file — f.close() at the end. Examiners do notice if this is missing.

Tracing questions: Show the file state after each write. Show the variable values after each read + split. Work through every iteration — don't skip.
