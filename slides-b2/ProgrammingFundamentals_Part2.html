// B2.2.1 · Static vs Dynamic Data Structures

How Memory Is Allocated and Resized

The key difference is whether the size is fixed at creation or can change during execution. This affects speed, memory use, and flexibility.

CharacteristicStaticDynamic
SizeFixed when the structure is created. Cannot grow or shrink.Can grow or shrink during execution.
Memory allocationAllocated once, in a contiguous block. Simple and predictable.Memory is allocated on demand. May use extra memory for pointers or metadata.
ResizingNot possible — you must know the maximum size in advance.Possible — elements can be added or removed at any time.
SpeedFast direct access by index (O(1) for random access).Varies — depends on the operation and structure type.
Best forData sets of a known, stable size — e.g. days of the week.Data sets whose size is unknown or changes — e.g. student enrolment.
Static example — Java array
int[] scores = new int[5];

Size is 5 from creation. Cannot add a 6th element without creating a new array.
Dynamic example — Python list / Java ArrayList
scores = [] (Python)
ArrayList<Integer> scores = new ArrayList<>(); (Java)

Size grows automatically as elements are added.

Exam tip: When comparing, always address three factors: memory, speed, and flexibility. A one-sentence answer ("dynamic is more flexible") will not earn full marks.

// B2.2.2 · 1D Arrays and Lists

Storing a Sequence of Related Values

A 1D array (Java) or 1D list (Python) stores many values under one name, each accessible by its index (starting at 0).

Python — dynamic list
scores = [72, 81, 90] # create scores.append(88) # add to end scores.remove(81) # remove by value first = scores[0] # index: 72 size = len(scores) # 3 # traverse for s in scores: print(s)
Java — static array + ArrayList
// static array (fixed size) int[] scores = {72, 81, 90}; int first = scores[0]; // 72 // ArrayList (dynamic) ArrayList<Integer> list = new ArrayList<>(); list.add(72); list.add(81); list.remove(Integer.valueOf(81)); // removes 81 int size = list.size(); // traverse for (int s : list) System.out.println(s);
OperationPython listJava ArrayList
Add to endlist.append(x)list.add(x)
Remove by valuelist.remove(x)list.remove(Integer.valueOf(x))
Access by indexlist[i]list.get(i)
Sizelen(list)list.size()
Insert at positionlist.insert(i, x)list.add(i, x)

Initialisation tip: When looking for a minimum value, initialise your variable to the first element of the list — not 0. The list might contain only positive numbers.

// B2.2.2 · 2D Arrays and Lists

Rows and Columns — Tables and Grids

A 2D structure is a list of lists (Python) or a 2D array (Java). Use two indices: first for the row, then for the column.

Python — 2D list
grades = [ ["Mina", 87, 91], # row 0 ["Leo", 78, 84], # row 1 ["Sara", 92, 95] # row 2 ] print(grades[1][0]) # "Leo" (row 1, col 0) print(grades[2][2]) # 95 (row 2, col 2) # traverse every cell for row in grades: for cell in row: print(cell, end=" ")
Java — 2D array
int[][] scores = new int[3][3]; // fill with user input for (int i = 0; i < 3; i++) { for (int j = 0; j < 3; j++) { scores[i][j] = scanner.nextInt(); } } // display for (int i = 0; i < 3; i++) { for (int j = 0; j < 3; j++) { System.out.print(scores[i][j] + " "); } System.out.println(); }
Use cases
Seating plans, game boards, mark sheets, spreadsheets, pixel grids.
Index rule
Row first, column second.
grades[2][1] = row 3, column 2.
Common mistake
Using only one loop for a 2D structure — you need nested loops to visit every cell.
// B2.2.3 · Stack — LIFO

Last In, First Out must include

A stack is a data structure where the most recently added item is always the first to be removed — like a stack of plates.

Four required operations
push — add an item to the top of the stack
pop — remove and return the top item
peek — read the top item without removing it
isEmpty — check whether the stack has no items
Python model
stack = [] stack.append("home") # push stack.append("settings") # push top = stack[-1] # peek → "settings" page = stack.pop() # pop → "settings" empty = len(stack) == 0 # isEmpty → False
Startstack: []
push "home"["home"] ← top
push "settings"["home", "settings"] ← top
peek → "settings"["home", "settings"] unchanged
pop → "settings"["home"] ← top
pop → "home"[] now empty

Real-world use: Browser back button, undo in a text editor, the call stack in a running program, expression evaluation.

// B2.2.3 · Stack — Performance & Memory

How Stack Operations Affect the Program

The IB requires you to explain performance and memory impact, not just name the operations.

OperationSpeedMemory note
pushO(1) — constant time. Adding to the top requires no shifting of existing elements.Consumes one more memory slot. If the stack is full (fixed-size), overflow occurs.
popO(1) — constant time. Removing from the top requires no shifting.Frees one memory slot. If the stack is empty, underflow (error) occurs.
peekO(1) — reads one value with no structural change.No memory change — just a read.
isEmptyO(1) — checks one condition.Always check before calling pop to prevent underflow.
Choosing a stack — when LIFO fits
Use a stack when the most recent item must always be processed first:

— Undo/redo in an editor
— Navigating back through browser history
— Checking matching brackets in code ( [ { } ] )
Overflow and underflow
Stack overflow: pushing to a full fixed-size stack.

Stack underflow: popping from an empty stack.

Always call isEmpty before pop to avoid underflow.
// B2.2.4 · Queue — FIFO

First In, First Out must include

A queue processes items in the order they arrive — like a ticket line. The first item to join is the first to be served.

Four required operations
enqueue — add an item to the rear of the queue
dequeue — remove and return the item at the front
front — inspect the front item without removing it
isEmpty — check whether the queue has no items
Python model (using list)
queue = [] queue.append("Ali") # enqueue queue.append("Bo") # enqueue queue.append("Cara") # enqueue front = queue[0] # front → "Ali" first = queue.pop(0) # dequeue → "Ali" empty = len(queue) == 0 # isEmpty → False
Startqueue: []
enqueue "Ali"front → ["Ali"] ← rear
enqueue "Bo"front → ["Ali", "Bo"] ← rear
enqueue "Cara"front → ["Ali", "Bo", "Cara"] ← rear
front → "Ali"unchanged
dequeue → "Ali"front → ["Bo", "Cara"] ← rear

Real-world use: Printer job queue, cafeteria line, CPU task scheduling, customer service tickets.

// B2.2.4 · Queue — Performance & Memory

How Queue Operations Affect the Program

OperationSpeed (Python list)Memory note
enqueueO(1) — appending to the end of a list is fast.Grows the queue by one element.
dequeueO(n) — removing from the front of a Python list shifts all elements left. For large queues, use collections.deque (O(1)).Frees one slot. Check isEmpty first to avoid errors on an empty queue.
frontO(1) — reads queue[0] with no structural change.No memory change.
isEmptyO(1) — one condition check.No memory change. Check before dequeue.
Choosing a queue — when FIFO fits
Use a queue when fairness matters — the longest-waiting item should be handled first:

— Print jobs sent to a printer
— Passengers boarding in order of check-in
— Network packets arriving at a router
Performance note for the exam
A standard Python list works as a queue at small scale. For large queues, collections.deque gives O(1) dequeue — because it avoids shifting every element on removal from the front.

Exam technique: Do not just name the structure. Justify it: "A stack is appropriate because the most recently opened page must be returned to first, which matches the LIFO order." Justification earns the marks.

Also consider lists: When you need indexed access — reading or updating any element by position — use a list or array, not a stack or queue.

// B2.2.2 · Traversal Patterns

Processing Every Element

Traversal means visiting every element in a structure. The IB expects you to construct traversal code and use it for calculations.

1D traversal — calculate average (Python)
scores = [72, 81, 90, 88] total = 0 for score in scores: total += score avg = total / len(scores) print("Average:", avg) # 82.75
Find max value (Python)
scores = [72, 81, 90, 88] max_score = scores[0] # start at first element for s in scores: if s > max_score: max_score = s print("Max:", max_score) # 90
2D traversal — sum all values (Python)
grades = [ [87, 91], [78, 84], [92, 95] ] total = 0 for row in grades: for cell in row: total += cell print("Sum:", total) # 527
2D traversal — Java
int total = 0; for (int i = 0; i < grades.length; i++) { for (int j = 0; j < grades[i].length; j++) { total += grades[i][j]; } } System.out.println(total);

Common mistake: Initialising a minimum/maximum tracker to 0 instead of the first element. If all values are positive and large, 0 will appear to be the minimum — which is wrong.
